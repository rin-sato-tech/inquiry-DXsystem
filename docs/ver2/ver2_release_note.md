# Ver.2 変更点まとめ

## 1. 目的

本ドキュメントは、社内問い合わせ管理システム Ver.2 で追加・変更した内容をまとめるものです。

Ver.1では、問い合わせの受付、記録、進捗管理、期限超過判定、集計、Tableau連携用CSV出力を実装しました。

Ver.2では、Ver.1の問い合わせ管理機能を拡張し、問い合わせ削減・自己解決支援につながる機能を追加しています。

## 2. Ver.2の位置づけ

Ver.2の位置づけは、問い合わせを「記録・管理する仕組み」から、問い合わせ削減や業務改善につなげる仕組みへ発展させることです。

具体的には、以下を重視しています。

- 対応漏れを防ぐ
- 確認往復を減らす
- よくある問い合わせをFAQ候補として蓄積する
- 依頼者が問い合わせ状況を確認できるようにする
- Ver.2追加機能の利用状況を集計・可視化する

## 3. 追加機能

| 機能                   | 内容                                                               |
| ---------------------- | ------------------------------------------------------------------ |
| 要対応アラート         | 期限超過、本日期限、期限間近、担当者未設定、情報待ち長期化を一覧化 |
| FAQ候補管理            | 完了済み問い合わせからFAQ候補を登録・更新・解除                    |
| FAQ候補CSV出力         | FAQ候補一覧をCSVとして出力                                         |
| カテゴリ別入力フォーム | 問い合わせカテゴリに応じた追加情報を入力                           |
| 依頼者向け確認画面     | 問い合わせIDまたは依頼者名で問い合わせ状況を確認                   |
| Ver.2追加集計          | アラート、FAQ候補、追加情報入力率、依頼者向け表示件数などを集計    |
| Tableau出力更新        | Ver.2追加列をTableau用CSVに出力                                    |
| DB移行処理             | 既存DBにVer.2追加カラムを追加                                      |
| Ver.2スモークテスト    | Ver.2追加機能の基本動作を確認                                      |

## 6. DB追加カラム

Ver.2では、既存の `inquiries` テーブルに以下のカラムを追加しました。

| カラム名                 | 型      | 初期値 | 内容                       |
| ------------------------ | ------- | -----: | -------------------------- |
| `faq_candidate`          | INTEGER |      0 | FAQ候補かどうか            |
| `faq_title`              | TEXT    | 空文字 | FAQタイトル                |
| `faq_answer`             | TEXT    | 空文字 | FAQ回答案                  |
| `additional_info`        | TEXT    | 空文字 | カテゴリ別追加情報         |
| `requester_visible`      | INTEGER |      1 | 依頼者向け画面に表示するか |
| `last_status_changed_at` | TEXT    | 空文字 | 最終ステータス変更日時     |

既存DBに対しては、以下で移行します。

```bash
python -m src.migrate_db
```

この処理は、既に存在するカラムを再追加しないようにしているため、複数回実行しても問題ありません。

## 10. 本格運用に向けた課題

Ver.2では、問い合わせ管理機能を拡張しましたが、本格運用に向けては以下が今後の課題です。

| 課題           | 内容                                                     |
| -------------- | -------------------------------------------------------- |
| ログイン認証   | 依頼者本人の問い合わせだけを表示するために必要           |
| 権限管理       | 依頼者、管理部、管理者で表示・更新範囲を分けるために必要 |
| FAQ公開ページ  | Ver.2ではFAQ候補管理までを対象としている                 |
| FAQ検索        | 問い合わせ前にFAQを検索できるようにする                  |
| メール通知     | 期限前、期限超過、ステータス変更時に通知する             |
| Slack連携      | 社内チャットへアラートや更新情報を通知する               |
| 添付ファイル   | スクリーンショットや証憑資料を添付できるようにする       |
| 操作履歴       | 誰がいつ何を更新したかを記録する                         |
| PostgreSQL移行 | 複数人利用や本格運用に向けてDBを移行する                 |

## 11. まとめ

Ver.2では、Ver.1の問い合わせ管理機能を拡張し、要対応アラート、FAQ候補管理、カテゴリ別入力フォーム、依頼者向け確認画面を追加しました。

これにより、問い合わせを記録・管理するだけでなく、対応漏れ防止、確認往復削減、ナレッジ化、依頼者側の進捗確認まで含めた簡易DXアプリに発展させています。
