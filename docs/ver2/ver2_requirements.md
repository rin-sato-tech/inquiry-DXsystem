# Ver.2 要件定義書

## 1. Ver.2開発の背景

初期版では、社内問い合わせ・依頼管理業務について、受付、記録、進捗管理、期限超過判定、集計、Tableau用CSV出力を実装した。

これにより、電話・口頭・メール・チャット・Excelに分散していた問い合わせを一元管理し、管理部の転記作業、進捗確認、期限確認、報告作成の負担を軽減できた。

一方で、現場利用を想定すると、以下のような改善余地が残っている。

- 期限超過や期限間近の問い合わせに、**画面を見に行かないと気づけない**
- **同じような問い合わせ**が繰り返し発生している
- 問い合わせ時点で**必要情報が不足**し、確認の往復が発生している
- 依頼者が**自分の問い合わせ状況を確認できない**
- 今後の業務改善に向けて、問い合わせ内容を**FAQ候補**として活用したい

## 2. Ver.2の目的

Ver.2では、初期版の問い合わせ管理機能を拡張し、以下を実現する。

1. 期限超過を未然に防ぐ
2. 確認の往復を減らす
3. 頻出問い合わせをFAQ候補として蓄積する
4. 依頼者が自分の問い合わせ状況を確認できるようにする
5. 問い合わせ管理から、問い合わせ削減・自己解決支援へ発展させる

## 3. Ver.2の実装範囲

Ver.2では、以下の機能を実装対象とする。

### 3.1 要対応アラート

未完了問い合わせのうち、対応優先度が高いものを抽出して表示する。

対象は以下とする。

- 期限超過
- 本日期限
- 期限間近
- 担当者未設定
- 情報待ち長期化

### 3.2 FAQ候補管理

完了済み問い合わせの中から、FAQ化できそうな問い合わせを管理部が選択できるようにする。

対象は以下とする。

- FAQ候補フラグの登録
- FAQタイトルの入力
- FAQ回答案の入力
- FAQ候補一覧の表示
- FAQ候補CSV出力

### 3.3 カテゴリ別入力フォーム

問い合わせカテゴリに応じて、追加の入力項目を表示する。

対象カテゴリの例は以下とする。

- PC・システム
- アカウント・権限
- 勤怠・労務
- 経費・請求
- 備品・設備

### 3.4 依頼者向け確認画面

依頼者が、自分の問い合わせ状況を確認できる画面を追加する。

検索条件は以下とする。

- 問い合わせID
- 依頼者名

表示項目は以下とする。

- 問い合わせID
- 受付日
- カテゴリ
- ステータス
- 担当者
- 希望期限
- 完了日
- 対応内容

## 4. Ver.2の対象外

Ver.2では、以下は実装対象外とする。

- 本格的なログイン認証
- 本格的な権限管理
- メール通知
- Slack通知
- 添付ファイルアップロード
- FAQ公開ページ
- AIによる自動分類
- 複数ユーザーによる同時更新を前提とした本格運用
- PostgreSQLなど外部DBへの移行

これらはVer.3以降の発展課題とする。

## 5. 利用者

Ver.2では、以下の利用者を想定する。

| 利用者       | 役割                                        |
| ------------ | ------------------------------------------- |
| 管理部担当者 | 問い合わせを確認・更新し、FAQ候補を登録する |
| 管理部長     | 期限超過や担当者別負荷を確認する            |
| 依頼者       | 自分の問い合わせ状況を確認する              |
| 経営層       | 問い合わせ傾向や改善効果を確認する          |

ただし、Ver.2ではログイン認証を実装しないため、依頼者向け確認画面はデモ用の簡易画面とする。

## 6. 期待効果

Ver.2により、以下の効果を期待する。

| 改善領域   | 期待効果                                         |
| ---------- | ------------------------------------------------ |
| 期限管理   | 期限超過・期限間近の問い合わせに気づきやすくなる |
| 入力品質   | カテゴリごとに必要情報を集めやすくなる           |
| 確認往復   | 管理部と依頼者の追加確認を減らす                 |
| ナレッジ化 | 頻出問い合わせをFAQ候補として蓄積できる          |
| 状況確認   | 依頼者からの進捗確認問い合わせを減らす           |
| 業務改善   | 問い合わせ傾向をもとに改善施策を検討できる       |

## 7. 技術方針

Ver.2でも、初期版と同じ技術スタックを継続する。

| 領域           | 技術         |
| -------------- | ------------ |
| 言語           | Python       |
| データ処理     | pandas       |
| DB             | SQLite       |
| 画面           | Streamlit    |
| 可視化         | Tableau      |
| データ連携     | CSV          |
| バージョン管理 | Git / GitHub |

Ver.2では新技術の導入よりも、既存アプリの拡張性、業務改善効果、画面設計、データ設計を重視する。

## 8. 完了条件

Ver.2は、以下を満たした時点で完了とする。

| 項目           | 完了条件                                         |
| -------------- | ------------------------------------------------ |
| 要対応アラート | 期限超過、期限間近、担当者未設定などを確認できる |
| FAQ候補管理    | 完了済み問い合わせをFAQ候補として登録できる      |
| FAQ候補出力    | FAQ候補を一覧表示・CSV出力できる                 |
| カテゴリ別入力 | カテゴリに応じた追加項目が表示される             |
| 追加情報保存   | 追加情報がDBに保存される                         |
| 依頼者向け確認 | 問い合わせIDまたは依頼者名で状況確認できる       |
| 集計更新       | アラート件数・FAQ候補件数が確認できる            |
| Tableau出力    | Ver.2追加列がCSVに含まれる                       |
| テスト         | スモークテストと手動テストが通る                 |
| ドキュメント   | README、操作手順、データ定義が更新されている     |
