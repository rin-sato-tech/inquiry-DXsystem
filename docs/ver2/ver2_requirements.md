# Ver.2 要件定義書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システム Ver.2 で追加・変更した要件を整理するものです。

ルート配下の `docs/requirements.md` は、Ver.1とVer.2を含めたシステム全体の要件定義書です。

本ドキュメントでは、Ver.1からVer.2へ拡張した差分を中心に整理します。

## 2. Ver.2開発の背景

Ver.1では、社内問い合わせ・依頼管理業務について、以下を実装しました。

- 問い合わせ一覧表示
- 新規登録
- ステータス更新
- 期限超過判定
- 基本集計
- Tableau用CSV出力

これにより、電話、口頭、メール、チャット、Excelに分散していた問い合わせを一元管理し、管理部の転記作業、進捗確認、期限確認、報告作成の負担を軽減できる構成にしました。

一方で、現場利用を想定すると、以下の改善余地が残っていました。

| 課題               | 内容                                            |
| ------------------ | ----------------------------------------------- |
| 対応漏れ           | 期限超過や担当者未設定に気づきにくい            |
| 確認往復           | 初回問い合わせ時点で必要情報が不足しやすい      |
| ナレッジ化不足     | 同じような問い合わせが繰り返し発生する          |
| 依頼者側の進捗不明 | 依頼者が自分の問い合わせ状況を確認しにくい      |
| 改善状況の把握不足 | FAQ候補や追加情報入力率などを数値で確認できない |

## 3. Ver.2の目的

Ver.2では、Ver.1の問い合わせ管理機能を拡張し、以下を実現します。

| 目的             | 内容                                                                         |
| ---------------- | ---------------------------------------------------------------------------- |
| 対応漏れ防止     | 要対応アラートにより、期限超過・本日期限・担当者未設定などを確認しやすくする |
| 確認往復削減     | カテゴリ別入力フォームにより、初回問い合わせ時点で必要情報を集める           |
| FAQ候補蓄積      | 完了済み問い合わせからFAQ候補を登録し、ナレッジ化につなげる                  |
| 依頼者向け確認   | 問い合わせIDまたは依頼者名で問い合わせ状況を確認できるようにする             |
| 改善状況の可視化 | アラート、FAQ候補、追加情報入力率などを集計・CSV出力できるようにする         |

Ver.2の位置づけは、問い合わせを「記録・管理する仕組み」から、問い合わせ削減・自己解決支援につながる仕組みへ発展させることです。

## 4. Ver.2の実装範囲

Ver.2で実装した機能は以下です。

### 4.1 要対応アラート

未完了問い合わせのうち、優先的に確認すべき問い合わせを抽出して表示します。

対象アラートは以下です。

| アラート       | 内容                             |
| -------------- | -------------------------------- |
| 期限超過       | 未完了、かつ希望期限を過ぎている |
| 本日期限       | 未完了、かつ希望期限が今日       |
| 期限間近       | 未完了、かつ希望期限が明日まで   |
| 担当者未設定   | 担当者が空欄                     |
| 情報待ち長期化 | 情報待ち状態が一定日数続いている |

### 4.2 FAQ候補管理

完了済み問い合わせの中から、FAQ化できそうな問い合わせを候補として管理します。

対象機能は以下です。

| 機能        | 内容                                   |
| ----------- | -------------------------------------- |
| FAQ候補登録 | 完了済み問い合わせをFAQ候補にする      |
| FAQ候補更新 | FAQタイトル・FAQ回答案を上書き更新する |
| FAQ候補解除 | FAQ候補フラグを外す                    |
| FAQ候補一覧 | 登録済みのFAQ候補を一覧表示する        |
| FAQ候補CSV  | FAQ候補をCSVとして出力する             |

Ver.2ではFAQ公開ページまでは実装せず、FAQ候補を蓄積する段階までを対象とします。

### 4.3 カテゴリ別入力フォーム

問い合わせカテゴリに応じて、追加の入力項目を表示します。

| カテゴリ         | 追加項目                                                 |
| ---------------- | -------------------------------------------------------- |
| PC・システム     | PC管理番号、発生時刻、エラー内容、再起動有無             |
| アカウント・権限 | 対象システム、対象フォルダ・対象機能、必要な権限、承認者 |
| 勤怠・労務       | 対象日、修正前時刻、修正後時刻、理由                     |
| 経費・請求       | 金額、取引先、対象月、添付書類有無                       |
| 備品・設備       | 備品名・設備名、数量、利用目的、希望日                   |

入力された内容は、`additional_info` に複数行テキストとして保存します。

### 4.4 依頼者向け確認画面

依頼者が問い合わせ状況を確認できる画面を追加します。

検索条件は以下です。

| 検索方法           | 内容                            |
| ------------------ | ------------------------------- |
| 問い合わせIDで検索 | `request_id` を指定して検索する |
| 依頼者名で検索     | `requester` を指定して検索する  |

検索UIはフォーム化し、Enterキーまたは検索ボタンで検索できるようにします。

表示対象は、`requester_visible == 1` の問い合わせです。

### 4.5 Ver.2追加集計

Ver.2では、以下の集計を追加します。

| 集計                  | 内容                                     |
| --------------------- | ---------------------------------------- |
| 要対応アラート件数    | 何らかのアラートに該当する問い合わせ件数 |
| アラート種別別件数    | 期限超過、本日期限、期限間近などの件数   |
| FAQ候補件数           | FAQ候補として登録された問い合わせ件数    |
| カテゴリ別FAQ候補件数 | FAQ候補をカテゴリ別に集計                |
| 追加情報入力件数      | カテゴリ別追加情報が入力されている件数   |
| 追加情報入力率        | 追加情報が入力されている割合             |
| 依頼者向け表示件数    | 依頼者向け画面に表示される問い合わせ件数 |
| 依頼者向け非表示件数  | 依頼者向け画面に表示しない問い合わせ件数 |

### 4.6 Tableau出力列の拡張

Tableau用CSVに、Ver.2追加機能に関する列を出力します。

| 列                        | 内容                                 |
| ------------------------- | ------------------------------------ |
| `has_alert`               | 要対応アラート対象か                 |
| `alert_type`              | アラート種別                         |
| `faq_candidate_int`       | FAQ候補かどうか                      |
| `has_additional_info_int` | カテゴリ別追加情報が入力されているか |
| `requester_visible_int`   | 依頼者向け画面に表示するか           |
| `requester_visible_label` | 依頼者向け表示区分                   |

## 5. Ver.2で追加したDB項目

Ver.2では、既存の `inquiries` テーブルに以下のカラムを追加します。

| カラム名                 | 型      | 初期値 | 内容                       |
| ------------------------ | ------- | -----: | -------------------------- |
| `faq_candidate`          | INTEGER |      0 | FAQ候補かどうか            |
| `faq_title`              | TEXT    | 空文字 | FAQタイトル                |
| `faq_answer`             | TEXT    | 空文字 | FAQ回答案                  |
| `additional_info`        | TEXT    | 空文字 | カテゴリ別追加情報         |
| `requester_visible`      | INTEGER |      1 | 依頼者向け画面に表示するか |
| `last_status_changed_at` | TEXT    | 空文字 | 最終ステータス変更日時     |

既存DBに対しては、`src/migrate_db.py` により不足カラムを追加します。

## 6. Ver.2の対象外

Ver.2では、以下は実装対象外とします。

| 対象外         | 理由                                                             |
| -------------- | ---------------------------------------------------------------- |
| ログイン認証   | Ver.2では依頼者向け確認画面をデモ画面として実装するため          |
| 権限管理       | 依頼者、管理部、管理者で表示範囲を分けるには追加設計が必要なため |
| メール通知     | ステータス変更や期限前通知は今後の拡張対象とするため             |
| Slack連携      | 社内通知ツールとの連携は今後の拡張対象とするため                 |
| FAQ公開ページ  | Ver.2ではFAQ候補管理までを対象とするため                         |
| FAQ検索        | 依頼者向けのFAQ検索機能はVer.3以降の課題とするため               |
| 添付ファイル   | スクリーンショットや証憑資料の管理には追加設計が必要なため       |
| 操作履歴       | 誰がいつ更新したかを記録する仕組みは今後の課題とするため         |
| AI自動分類     | データ量が少ない段階では対象外とするため                         |
| PostgreSQL移行 | Ver.2ではSQLiteを継続利用するため                                |

## 7. 利用者

Ver.2で想定する利用者は以下です。

| 利用者       | 主な利用内容                                                      |
| ------------ | ----------------------------------------------------------------- |
| 管理部担当者 | アラート確認、問い合わせ更新、FAQ候補登録、カテゴリ別追加情報確認 |
| 管理部責任者 | アラート件数、担当者別負荷、FAQ候補件数、追加情報入力率の確認     |
| 依頼者       | 問い合わせIDまたは依頼者名で問い合わせ状況を確認                  |
| 経営層       | 問い合わせ傾向、削減効果、改善状況の確認                          |

ただし、Ver.2ではログイン認証を実装しないため、依頼者向け確認画面はデモ用の簡易画面とします。

## 8. 期待効果

Ver.2により、以下の効果を期待します。

| 改善領域   | 期待効果                                                   |
| ---------- | ---------------------------------------------------------- |
| 期限管理   | 期限超過、本日期限、期限間近の問い合わせに気づきやすくなる |
| 担当者管理 | 担当者未設定の問い合わせを見つけやすくなる                 |
| 入力品質   | カテゴリごとに必要情報を集めやすくなる                     |
| 確認往復   | 管理部と依頼者の追加確認を減らす                           |
| ナレッジ化 | 頻出問い合わせをFAQ候補として蓄積できる                    |
| 状況確認   | 依頼者からの進捗確認問い合わせを減らす                     |
| 業務改善   | アラート、FAQ候補、追加情報入力率を数値で確認できる        |

## 9. 技術方針

Ver.2でも、Ver.1と同じ技術スタックを継続します。

| 領域           | 技術         |
| -------------- | ------------ |
| 言語           | Python       |
| データ処理     | pandas       |
| DB             | SQLite       |
| 画面           | Streamlit    |
| 可視化         | Tableau      |
| データ連携     | CSV          |
| バージョン管理 | Git / GitHub |

Ver.2では、新技術の導入よりも、既存アプリの拡張性、業務改善効果、画面設計、データ設計を重視します。

## 10. 完了条件

Ver.2は、以下を満たした時点で完了とします。

| 項目           | 完了条件                                                                   |
| -------------- | -------------------------------------------------------------------------- |
| DB移行         | Ver.2追加カラムを既存DBに追加できる                                        |
| 要対応アラート | 期限超過、本日期限、期限間近、担当者未設定などを確認できる                 |
| FAQ候補管理    | 完了済み問い合わせをFAQ候補として登録・更新・解除できる                    |
| FAQ候補出力    | FAQ候補を一覧表示・CSV出力できる                                           |
| カテゴリ別入力 | カテゴリに応じた追加項目が表示される                                       |
| 追加情報保存   | カテゴリ別追加情報がDBに保存される                                         |
| 依頼者向け確認 | 問い合わせIDまたは依頼者名で状況確認できる                                 |
| 集計更新       | アラート件数、FAQ候補件数、追加情報入力率などを確認できる                  |
| Tableau出力    | Ver.2追加列がCSVに含まれる                                                 |
| テスト         | Ver.1スモークテスト、Ver.2スモークテスト、手動確認が通る                   |
| ドキュメント   | README、要件定義、画面設計、データ定義、操作手順、保守手順が更新されている |

## 11. 今後の拡張方針

Ver.3以降では、以下を検討します。

- ログイン認証
- 権限管理
- FAQ公開ページ
- FAQ検索
- メール通知
- Slack連携
- 添付ファイル対応
- 操作履歴
- ステータス履歴
- PostgreSQL移行
- Flask / Django / FastAPIなどを用いたWebアプリ化
- TableauダッシュボードのVer.2対応
