# Ver.2 画面設計書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システム Ver.2 で追加・変更した画面設計を整理するものです。

ルート配下の `docs/screen_design.md` は、Ver.1とVer.2を含めたシステム全体の画面設計書です。

本ドキュメントでは、Ver.2で追加した画面と、既存画面から変更した点を中心に整理します。

## 2. Ver.2画面設計の方針

Ver.2では、Ver.1の画面構成を大きく作り替えるのではなく、既存画面を拡張しながら新しい画面を追加します。

Ver.1の主な画面は以下です。

- 問い合わせ一覧
- 新規登録
- ステータス更新
- 集計・CSV出力

Ver.2では、以下を追加・拡張します。

- 要対応アラート
- FAQ候補管理
- 依頼者向け確認
- 新規登録画面のカテゴリ別入力フォーム
- ステータス更新画面のカテゴリ別追加情報表示
- 集計・CSV出力画面のVer.2追加集計
- Tableau出力列の拡張

## 3. Ver.2のタブ構成

Ver.2時点のタブ構成は以下です。

| タブ           | 目的                                                     |
| -------------- | -------------------------------------------------------- |
| 要対応アラート | 期限超過・本日期限・期限間近・担当者未設定などを確認する |
| 問い合わせ一覧 | 全問い合わせを一覧・絞り込み表示する                     |
| 新規登録       | 問い合わせを登録する                                     |
| ステータス更新 | 担当者、ステータス、対応内容、作業時間を更新する         |
| FAQ候補管理    | 完了済み問い合わせからFAQ候補を登録・更新・解除する      |
| 依頼者向け確認 | 依頼者が問い合わせ状況を確認する                         |
| 集計・CSV出力  | 集計結果とTableau用CSV出力を確認する                     |

## 4. 要対応アラート画面

### 4.1 目的

管理部担当者が、優先的に確認すべき問い合わせをすぐ把握できるようにします。

### 4.2 表示するKPI

| KPI                | 内容                                     |
| ------------------ | ---------------------------------------- |
| アラート対象件数   | 何らかのアラートに該当する問い合わせ件数 |
| 期限超過件数       | 未完了で希望期限を過ぎている件数         |
| 本日期限件数       | 未完了で希望期限が今日の件数             |
| 期限間近件数       | 未完了で希望期限が明日までの件数         |
| 担当者未設定件数   | 担当者が空欄の件数                       |
| 情報待ち長期化件数 | 情報待ち状態が一定日数以上続いている件数 |

### 4.3 絞り込み

| フィルタ     | 内容                                                               |
| ------------ | ------------------------------------------------------------------ |
| アラート種別 | すべて、期限超過、本日期限、期限間近、担当者未設定、情報待ち長期化 |

### 4.4 表示項目

| 項目           | 内容           |
| -------------- | -------------- |
| `request_id`   | 問い合わせID   |
| `request_date` | 受付日         |
| `requester`    | 依頼者         |
| `department`   | 部署           |
| `category`     | カテゴリ       |
| `priority`     | 優先度         |
| `due_date`     | 希望期限       |
| `assignee`     | 担当者         |
| `status`       | ステータス     |
| `alert_type`   | アラート種別   |
| `detail`       | 問い合わせ内容 |

## 5. 問い合わせ一覧画面の変更点

### 5.1 目的

問い合わせ一覧画面は、登録済み問い合わせの全体状況を確認する画面です。

Ver.2では、Ver.1の一覧表示に加えて、カテゴリ別追加情報やアラート関連情報を確認できるようにします。

### 5.2 Ver.2で追加する表示候補

| 項目                | 内容                 |
| ------------------- | -------------------- |
| `additional_info`   | カテゴリ別追加情報   |
| `has_alert`         | 要対応アラート対象か |
| `alert_type`        | アラート種別         |
| `faq_candidate`     | FAQ候補かどうか      |
| `requester_visible` | 依頼者向け表示対象か |

ただし、一覧表に表示する列が増えすぎると見づらくなるため、重要列を中心に表示し、詳細は各画面で確認する方針とします。

### 5.3 画面設計上の注意点

- 一覧画面は全体確認を目的とする
- FAQ候補の編集はFAQ候補管理画面に分ける
- 依頼者向け表示制御はVer.2では基本的にデータ側のフラグとして扱う
- 詳細な追加情報は必要に応じて表示する

## 6. 新規登録画面の変更点

### 6.1 目的

新規登録画面は、新しい問い合わせを登録する画面です。

Ver.2では、問い合わせカテゴリに応じて追加項目を表示し、初回登録時点で必要情報を集めやすくします。

### 6.2 カテゴリ別入力フォーム

カテゴリを選択すると、以下の追加項目を表示します。

| カテゴリ         | 追加項目                                                 |
| ---------------- | -------------------------------------------------------- |
| PC・システム     | PC管理番号、発生時刻、エラー内容、再起動有無             |
| アカウント・権限 | 対象システム、対象フォルダ・対象機能、必要な権限、承認者 |
| 勤怠・労務       | 対象日、修正前時刻、修正後時刻、理由                     |
| 経費・請求       | 金額、取引先、対象月、添付書類有無                       |
| 備品・設備       | 備品名・設備名、数量、利用目的、希望日                   |

### 6.3 保存方法

カテゴリ別追加項目は、`additional_info` に複数行テキストとして保存します。

例：

```text
PC管理番号: PC-014
発生時刻: 09:30
エラー内容: 販売管理システム起動時に認証エラー
再起動有無: 実施済み
```

Ver.2では、DB構造を複雑にしすぎないため、カテゴリ別詳細項目を個別カラムや別テーブルに分けず、1つのテキスト項目に集約します。

## 7. ステータス更新画面の変更点

### 7.1 目的

ステータス更新画面は、問い合わせの担当者、ステータス、完了日、対応内容、作業時間などを更新する画面です。

Ver.2では、更新対象の問い合わせに登録されたカテゴリ別追加情報を確認できるようにします。

### 7.2 表示する情報

問い合わせを選択した後、以下を表示します。

| 項目               | 内容               |
| ------------------ | ------------------ |
| `request_id`       | 問い合わせID       |
| `requester`        | 依頼者             |
| `department`       | 部署               |
| `category`         | カテゴリ           |
| `subcategory`      | サブカテゴリ       |
| `detail`           | 問い合わせ内容     |
| `missing_info`     | 不足情報・確認事項 |
| `additional_info`  | カテゴリ別追加情報 |
| `due_date`         | 希望期限           |
| `status`           | 現在のステータス   |
| `response_summary` | 現在の対応内容     |

`additional_info` がない場合は、カテゴリ別追加情報が登録されていない旨を表示します。

### 7.3 更新する項目

| 項目                      | 内容               |
| ------------------------- | ------------------ |
| `assignee`                | 担当者             |
| `status`                  | ステータス         |
| `completed_date`          | 完了日             |
| `response_summary`        | 対応内容           |
| `record_issue`            | 記録・管理上の問題 |
| `management_minutes`      | 管理作業時間       |
| `actual_response_minutes` | 実対応時間         |

FAQ候補の登録・更新・解除は、ステータス更新画面ではなくFAQ候補管理画面で行います。

## 8. FAQ候補管理画面

### 8.1 目的

完了済み問い合わせの中から、FAQ化できそうな問い合わせを管理する画面です。

Ver.2では、FAQ公開ページではなく、FAQ候補を蓄積・整理することを目的とします。

### 8.2 対象データ

FAQ候補管理画面では、ステータスが完了の問い合わせを対象とします。

```text
status == "完了"
```

### 8.3 表示項目

問い合わせを選択した後、以下を表示します。

| 項目               | 内容               |
| ------------------ | ------------------ |
| `request_id`       | 問い合わせID       |
| `category`         | カテゴリ           |
| `subcategory`      | サブカテゴリ       |
| `detail`           | 問い合わせ内容     |
| `additional_info`  | カテゴリ別追加情報 |
| `response_summary` | 対応内容           |
| `completed_date`   | 完了日             |
| `faq_candidate`    | FAQ候補かどうか    |

問い合わせ詳細の表示順は以下とします。

```text
問い合わせ内容
カテゴリ別追加情報
対応内容
完了日
FAQ候補状態メッセージ
```

FAQ候補状態は、以下のように表示します。

| 状態            | 表示                                        |
| --------------- | ------------------------------------------- |
| FAQ候補である   | この問い合わせは現在FAQ候補です。           |
| FAQ候補ではない | この問い合わせはまだFAQ候補ではありません。 |

### 8.4 入力項目

| 項目         | 内容                            |
| ------------ | ------------------------------- |
| `faq_title`  | FAQとして整理する場合のタイトル |
| `faq_answer` | 依頼者向けに整理した回答案      |

### 8.5 操作

| 操作                     | 内容                                                   |
| ------------------------ | ------------------------------------------------------ |
| FAQ候補として保存・更新  | `faq_candidate` を1にし、FAQタイトル・回答案を保存する |
| FAQ候補から外す          | `faq_candidate` を0にする                              |
| FAQ候補CSVをダウンロード | FAQ候補一覧をCSVとして出力する                         |

FAQ候補から外しても、FAQタイトル・FAQ回答案は下書きとして残します。

## 9. 依頼者向け確認画面

### 9.1 目的

依頼者が、自分の問い合わせ状況を確認できるようにする画面です。

Ver.2ではデモ画面として実装し、本格運用ではログイン認証・権限管理が必要です。

### 9.2 検索条件

| 検索方法     | 内容                            |
| ------------ | ------------------------------- |
| 問い合わせID | `request_id` を入力して検索する |
| 依頼者名     | `requester` を入力して検索する  |

検索UIは `st.form()` を使用し、Enterキーまたは検索ボタンで検索できるようにします。

### 9.3 表示対象

依頼者向け確認画面では、`requester_visible == 1` の問い合わせを表示します。

`requester_visible == 0` の問い合わせは表示しません。

### 9.4 表示する項目

| 項目               | 内容                       |
| ------------------ | -------------------------- |
| `request_id`       | 問い合わせID               |
| `request_date`     | 受付日                     |
| `requester`        | 依頼者                     |
| `department`       | 部署                       |
| `category`         | カテゴリ                   |
| `subcategory`      | サブカテゴリ               |
| `detail`           | 問い合わせ内容             |
| `additional_info`  | カテゴリ別追加情報         |
| `status`           | ステータス                 |
| `assignee`         | 担当者                     |
| `due_date`         | 希望期限                   |
| `completed_date`   | 完了日                     |
| `response_summary` | 管理部からの回答・対応内容 |

### 9.5 表示しない項目

| 項目                      | 理由                        |
| ------------------------- | --------------------------- |
| `management_minutes`      | 管理部内部の作業時間のため  |
| `actual_response_minutes` | 管理部内部の作業時間のため  |
| `record_issue`            | 管理上の内部メモのため      |
| `faq_candidate`           | FAQ管理用の内部フラグのため |
| `faq_title`               | FAQ候補管理用の下書きのため |
| `faq_answer`              | FAQ候補管理用の下書きのため |

### 9.6 注意書き

画面上には、以下の注意書きを表示します。

```text
この画面は、依頼者が自分の問い合わせ状況を確認する想定のデモ画面です。
本格運用ではログイン認証・権限管理が必要です。
```

## 10. 集計・CSV出力画面の変更点

### 10.1 目的

問い合わせ状況を集計し、Tableau用CSVを出力する画面です。

Ver.2では、基本集計に加えて、アラート、FAQ候補、追加情報入力率、依頼者向け表示件数を確認できるようにします。

### 10.2 Ver.2追加集計

| 集計                     | 内容                                     |
| ------------------------ | ---------------------------------------- |
| 要対応アラート件数       | 何らかのアラートに該当する問い合わせ件数 |
| アラート種別別件数       | 期限超過、本日期限、期限間近などの件数   |
| FAQ候補件数              | FAQ候補として登録された問い合わせ件数    |
| カテゴリ別FAQ候補件数    | FAQ候補をカテゴリ別に集計                |
| 追加情報入力件数         | カテゴリ別追加情報が入力されている件数   |
| 追加情報入力率           | 追加情報が入力されている割合             |
| カテゴリ別追加情報入力率 | カテゴリごとの追加情報入力率             |
| 依頼者向け表示件数       | 依頼者向け画面に表示される問い合わせ件数 |
| 依頼者向け非表示件数     | 依頼者向け画面に表示しない問い合わせ件数 |

### 10.3 Tableau出力に追加する項目

| 列名                      | 内容                       |
| ------------------------- | -------------------------- |
| `additional_info`         | カテゴリ別追加情報         |
| `has_additional_info`     | 追加情報が入力されているか |
| `has_additional_info_int` | 追加情報ありなら1          |
| `faq_candidate`           | FAQ候補フラグ              |
| `faq_candidate_int`       | FAQ候補なら1               |
| `faq_title`               | FAQタイトル                |
| `faq_answer`              | FAQ回答案                  |
| `requester_visible`       | 依頼者向け表示フラグ       |
| `requester_visible_int`   | 依頼者向け表示なら1        |
| `requester_visible_label` | 依頼者向け表示区分         |
| `alert_overdue`           | 期限超過アラート           |
| `alert_due_today`         | 本日期限アラート           |
| `alert_due_soon`          | 期限間近アラート           |
| `alert_unassigned`        | 担当者未設定アラート       |
| `alert_info_waiting_long` | 情報待ち長期化アラート     |
| `has_alert`               | 何らかのアラート対象か     |
| `alert_type`              | アラート種別               |

## 11. 画面設計上の注意点

Ver.2では機能が増えるため、画面が複雑になりすぎないようにします。

特に以下に注意します。

| 注意点                       | 内容                                                     |
| ---------------------------- | -------------------------------------------------------- |
| タブの役割分担               | アラート確認、更新、FAQ管理、集計の役割を分ける          |
| 一覧の列数                   | 一覧表に表示する列を増やしすぎない                       |
| 詳細情報の表示               | 長い情報は展開表示や詳細表示に寄せる                     |
| 管理部向けと依頼者向けの分離 | 依頼者向け画面では内部情報を表示しない                   |
| FAQ管理の分離                | FAQ候補登録はFAQ候補管理画面に集約する                   |
| 本格運用との差分             | 依頼者向け画面には認証・権限管理が必要である旨を明示する |
