# 発展課題

## 1. 初期版の位置づけ

本システムは、社内問い合わせ管理のMVPとして作成した。
初期版では、問い合わせの受付、記録、進捗管理、期限超過判定、集計、Tableau連携を対象とした。
一方で、実運用を想定すると、追加で検討すべき課題がある。

## 2. 機能面の発展課題

| 課題             | 内容                                             |
| ---------------- | ------------------------------------------------ |
| ログイン認証     | 利用者を識別し、操作履歴を残す                   |
| 権限管理         | 依頼者、管理部、管理者で操作範囲を分ける         |
| 通知機能         | 期限前や期限超過時に担当者へ通知する             |
| メール連携       | 問い合わせ登録時や更新時にメール通知する         |
| コメント履歴     | 対応過程を履歴として残す                         |
| 添付ファイル対応 | スクリーンショットや申請書を添付できるようにする |
| 承認フロー       | 権限付与や契約書確認などに承認ステップを設ける   |

## 3. 分析面の発展課題

| 課題             | 内容                                     |
| ---------------- | ---------------------------------------- |
| FAQ候補抽出      | 頻出問い合わせを抽出し、ナレッジ化する   |
| カテゴリ自動分類 | 問い合わせ内容からカテゴリを推定する     |
| 対応時間予測     | カテゴリや優先度から対応時間を予測する   |
| 担当者負荷予測   | 問い合わせ傾向から負荷集中を予測する     |
| 月次推移分析     | 問い合わせ件数や作業時間の推移を分析する |

## 4. 技術面の発展課題

| 課題                           | 内容                                    |
| ------------------------------ | --------------------------------------- |
| SQLiteからPostgreSQL等への移行 | 複数人利用や同時更新に強くする          |
| Streamlit以外の画面            | Flask、FastAPI、Reactなどへの発展       |
| API化                          | 問い合わせ登録・更新処理をAPIとして分離 |
| Docker化                       | 環境構築を簡単にする                    |
| クラウド配置                   | 社内利用を想定したデプロイを行う        |

## 5. ノーコード・ローコードとの比較

実務上は、以下のような選択肢も考えられる。

- Googleフォーム
- Googleスプレッドシート
- AppSheet
- kintone
- Microsoft Forms
- Power Apps

受付だけであればGoogleフォームでも十分である。
ただし、本プロジェクトでは、問い合わせ受付後の進捗管理、期限超過判定、作業時間集計、Tableau連携までを対象としたため、Python、SQLite、Streamlitを用いて実装した。

## 6. 優先順位

今後拡張するなら、優先順位は以下が妥当である。

| 優先度 | 課題           | 理由                                   |
| ------ | -------------- | -------------------------------------- |
| 高     | FAQ候補抽出    | 問い合わせ件数そのものの削減につながる |
| 高     | 通知機能       | 期限超過の予防につながる               |
| 中     | 権限管理       | 実運用時には必要                       |
| 中     | コメント履歴   | 対応過程の透明化につながる             |
| 低     | AI分類         | データ量が増えてから検討する           |
| 低     | 本格承認フロー | 仕様が複雑化するため後回し             |

## Ver.2で対応したこと

Ver.2では、Ver.1の問い合わせ管理機能を拡張し、以下を実装した。

| 項目                   | 内容                                                 |
| ---------------------- | ---------------------------------------------------- |
| 要対応アラート         | 期限超過・本日期限・期限間近・担当者未設定などを表示 |
| FAQ候補管理            | 完了済み問い合わせからFAQ候補を蓄積                  |
| カテゴリ別入力フォーム | カテゴリに応じた追加情報を登録                       |
| 依頼者向け確認画面     | 問い合わせID・依頼者名で状況確認                     |
| Ver.2集計              | アラート、FAQ候補、追加情報入力率などを集計          |
| Tableau出力拡張        | Ver.2追加列をCSVに出力                               |
