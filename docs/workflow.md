# 業務フロー設計書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システムの業務フローを整理するものです。

DX化前の問い合わせ対応業務と、システム導入後の業務フローを比較し、どの工程を効率化するかを明確にします。

Ver.1では、問い合わせの受付、記録、進捗管理、期限超過判定、集計、Tableau連携を中心に業務フローを整理しました。
Ver.2では、Ver.1の問い合わせ管理フローを拡張し、以下を業務フローに追加しています。

- 要対応アラートによる優先確認
- カテゴリ別入力フォームによる確認往復削減
- FAQ候補管理によるナレッジ化
- 依頼者向け確認による進捗確認負担の削減
- Ver.2追加集計による改善状況の可視化

## 2. DX化前の業務フロー

DX化前は、社員からの問い合わせが電話、口頭、メール、チャット、Excelなど複数経路で管理部に届いています。

```text
社員：電話・口頭・メール・チャット・Excelで問い合わせ
管理部担当者が内容を確認
必要情報が不足していれば個別に確認
  ↓
Excel台帳へ手作業で転記
担当者を設定
  ↓
対応状況を手作業で更新
期限超過や未対応を目視で確認
  ↓
週次・月次でExcel集計
管理部長・経営層へ報告
```

## 3. DX化前の問題点

| 工程       | 問題                                                            |
| ---------- | --------------------------------------------------------------- |
| 受付       | 受付経路が分散しており、問い合わせの所在が分かりにくい          |
| 内容確認   | 初回問い合わせ時に必要情報が不足しやすい                        |
| 転記       | 電話・口頭・チャットの内容を後からExcelへ入力する必要がある     |
| 進捗管理   | ステータス更新が後回しになりやすい                              |
| 期限管理   | Excelを目視確認しないと期限超過に気づきにくい                   |
| 担当者管理 | 担当者未設定の問い合わせが埋もれやすい                          |
| 依頼者確認 | 依頼者が自分の問い合わせ状況を確認しにくい                      |
| 報告作成   | 週次・月次で手作業集計が必要                                    |
| 属人化     | PC・システム、アカウント・権限の対応が特定担当者に集中しやすい  |
| ナレッジ化 | よくある問い合わせがFAQ化されず、同種の問い合わせが繰り返される |

## 4. Ver.1のDX化後業務フロー

Ver.1では、問い合わせをStreamlitフォームから登録し、SQLiteで一元管理する流れに変更しました。

```text id="c7cm9l"
社員または管理部担当者：Streamlitの問い合わせ登録フォームから入力
SQLiteに問い合わせデータを保存
  ↓
管理部が問い合わせ一覧画面で確認
担当者・ステータス・対応内容を更新
  ↓
Pythonで期限超過・対応日数を自動判定
Streamlitで未対応・期限超過・担当者別件数を確認
Tableau用CSVを出力
Tableauで報告用ダッシュボードを確認
```

## 5. Ver.2のDX化後業務フロー

Ver.2では、Ver.1の問い合わせ管理フローに、要対応アラート、カテゴリ別入力フォーム、FAQ候補管理、依頼者向け確認を追加します。

```text
社員または管理部担当者：カテゴリ別入力フォームから問い合わせ内容を登録
SQLiteに問い合わせデータを保存
  ↓
管理部が要対応アラートを確認
期限超過・本日期限・担当者未設定などを優先確認
  ↓
管理部が問い合わせ一覧で全体状況を確認
担当者・ステータス・対応内容を更新
  ↓
依頼者は依頼者向け確認画面で状況を確認
  ↓
完了済み問い合わせをFAQ候補として整理
Pythonで基本集計・Ver.2追加集計を実行
Tableau用CSVを出力
Tableauで報告用ダッシュボードを確認
```

## 6. Ver.2で追加した業務フロー

### 6.1 要対応アラート確認フロー

要対応アラートは、管理部が優先的に確認すべき問い合わせを見つけるためのフローです。

```text
管理部担当者：要対応アラート画面を開く
アラート対象件数を確認
期限超過・本日期限・期限間近・担当者未設定・情報待ち長期化を確認
  ↓
必要に応じてアラート種別で絞り込み
対象問い合わせIDを確認
  ↓
ステータス更新画面で担当者・ステータス・対応内容を更新
```

このフローにより、Excelを目視確認していた期限管理を、アラート中心の確認に置き換えます。

### 6.2 カテゴリ別入力フォーム利用フロー

カテゴリ別入力フォームは、初回問い合わせ時点で必要情報を集めるためのフローです。

```text
管理部担当者：新規登録画面を開く
問い合わせカテゴリを選択
  ↓
カテゴリに応じた追加項目が表示される
PC管理番号、対象日、金額、承認者などを入力
問い合わせ内容と追加情報を登録
  ↓
ステータス更新画面やFAQ候補管理画面で追加情報を確認
```

このフローにより、管理部と依頼者の確認往復を減らします。

### 6.3 FAQ候補管理フロー

FAQ候補管理は、完了済み問い合わせをナレッジ化するためのフローです。

```text
管理部担当者：FAQ候補管理画面を開く
完了済み問い合わせを選択
  ↓
問い合わせ内容・カテゴリ別追加情報・対応内容・完了日を確認
FAQタイトル・FAQ回答案を入力
FAQ候補として保存・更新
  ↓
FAQ候補一覧やFAQ候補CSVで確認
```

Ver.2では、FAQ公開ページまでは実装せず、FAQ候補を蓄積する段階までを対象とします。

### 6.4 依頼者向け確認フロー

依頼者向け確認は、依頼者が問い合わせ状況を確認するためのフローです。

```text
依頼者：依頼者向け確認画面を開く
問い合わせIDまたは依頼者名で検索
  ↓
ステータス・担当者・希望期限・完了日を確認
必要に応じて詳細を確認
```

Ver.2ではデモ画面として実装しています。本格運用では、ログイン認証と権限管理により、本人の問い合わせだけを表示する必要があります。

### 6.5 Ver.2集計・CSV出力フロー

Ver.2では、基本集計に加えて、アラート、FAQ候補、追加情報入力率、依頼者向け表示件数も集計します。

```text
管理部担当者または管理部責任者：集計・CSV出力画面を開く
基本KPI・Ver.2追加KPIを確認
  ↓
Tableau用CSVを出力
Tableauでダッシュボードを確認
  ↓
月次報告や改善状況の確認に利用
```

## 7. DX化後に改善される点

| 観点       | DX化前                                    | Ver.1                  | Ver.2                                     |
| ---------- | ----------------------------------------- | ---------------------- | ----------------------------------------- |
| 受付       | 電話・口頭・メール・チャット・Excelに分散 | フォーム登録に統一     | カテゴリ別入力フォームで必要情報も収集    |
| 記録       | Excel台帳に手入力                         | SQLiteに保存           | FAQ候補・追加情報・依頼者表示フラグも管理 |
| 必要情報   | 後から個別確認                            | フォーム項目で取得     | カテゴリ別追加項目で確認往復を削減        |
| 進捗管理   | Excel上で手作業更新                       | Streamlitで更新        | 依頼者向け確認画面でも状況確認            |
| 期限管理   | 目視確認                                  | 期限超過を自動判定     | 要対応アラートで優先確認                  |
| 担当者管理 | 担当者未設定が埋もれやすい                | 一覧で確認             | 担当者未設定アラートで検知                |
| 報告作成   | Excelで手集計                             | CSV出力・Tableau可視化 | Ver.2追加機能も集計対象にする             |
| 属人化     | 担当者の記憶に依存                        | データとして共有       | FAQ候補としてナレッジ化                   |
| 依頼者確認 | 管理部へ個別確認                          | 管理部側で状況確認     | 依頼者向け確認画面で検索可能              |

## 8. 管理部担当者の業務フロー

管理部担当者は、日次で以下の流れを想定します。

```text
出社後、要対応アラート画面を確認
期限超過・本日期限・担当者未設定を確認
問い合わせ一覧で全体状況を確認
  ↓
新規問い合わせを登録
  ↓
担当者・ステータス・対応内容を更新
  ↓
完了した問い合わせをFAQ候補として整理
  ↓
必要に応じて集計・CSV出力を確認
```

この流れにより、管理部担当者は、問い合わせ全体を確認しながら優先度の高いものから対応できます。

## 9. 管理部責任者の業務フロー

管理部責任者は、週次・月次で以下の流れを想定します。

```text
集計・CSV出力画面を確認
問い合わせ件数、未完了件数、期限超過件数を確認
担当者別件数・作業時間を確認
アラート件数を確認
FAQ候補件数や追加情報入力率を確認
  ↓
Tableau用CSVを出力
Tableauダッシュボードで月次傾向を確認
  ↓
管理部長・経営層へ報告
```

Ver.2では、問い合わせ件数や作業時間だけでなく、アラート、FAQ候補、追加情報入力率なども確認対象にします。

## 10. 依頼者の業務フロー

依頼者は、問い合わせ登録後に以下の流れで状況を確認します。

```text
問い合わせを依頼
問い合わせIDを控える
  ↓
依頼者向け確認画面を開く
問い合わせIDまたは依頼者名で検索
ステータス・担当者・希望期限・完了日を確認
  ↓
必要に応じて管理部へ追加連絡
```

このフローにより、依頼者が管理部へ個別に進捗確認する回数を減らすことを想定します。

## 11. 原田の繁忙日の業務例

原田は、PC・システム、アカウント・権限に関する問い合わせを主に担当します。
通常日は1日1〜3件程度を担当しますが、月初、月末、週次報告日、システム不具合発生日には5〜6件程度に増える想定です。
繁忙日には、以下のような問い合わせが重なります。

| 時刻  | 問い合わせ                     | 経路     | 状態     |
| ----- | ------------------------------ | -------- | -------- |
| 9:20  | 共有フォルダ権限付与           | メール   | 完了     |
| 9:45  | PCが起動しない                 | 電話     | 対応中   |
| 10:20 | 販売管理システムのエラー       | チャット | 完了     |
| 11:30 | 複合機で印刷できない           | 口頭     | 完了     |
| 13:30 | 勤怠システムにログインできない | 電話     | 完了     |
| 14:10 | 新入社員アカウント作成         | メール   | 情報待ち |

DX化前は、このような問い合わせ対応と台帳更新・週次集計に約4時間を要する想定です。
ただし、DX化で削減できるのは、実対応そのものではなく、主に以下です。

- 受付確認
- 転記
- ステータス更新
- 期限確認
- 担当者未設定確認
- 報告用集計
- FAQ候補整理
- 進捗確認への個別回答

## 12. Ver.2導入後の原田の業務例

Ver.2導入後は、原田の繁忙日の確認フローは以下のように変わります。

```text id="bvadug"
要対応アラートで本日期限・担当者未設定を確認
PC・システム、アカウント・権限の問い合わせを優先確認
カテゴリ別追加情報を確認し、必要情報が揃っているか判断
  ↓
対応後、ステータスと対応内容を更新
完了済みで頻出内容のものをFAQ候補として登録
  ↓
週次報告では集計・CSV出力画面から件数と作業時間を確認
```

これにより、原田はExcel台帳の目視確認や手作業集計に使っていた時間を減らし、実対応や改善活動に時間を回しやすくなります。

## 13. 業務フロー上の実装対象外

Ver.2では、以下の業務フローは実装対象外とします。

| 対象外         | 理由                                                             |
| -------------- | ---------------------------------------------------------------- |
| ログイン認証   | Ver.2ではデモ用の依頼者向け画面として実装するため                |
| 権限管理       | 依頼者、管理部、管理者で表示範囲を分けるには追加設計が必要なため |
| メール通知     | ステータス変更や期限前通知は今後の拡張対象とするため             |
| Slack連携      | 社内通知との連携は今後の拡張対象とするため                       |
| FAQ公開ページ  | Ver.2ではFAQ候補管理までを対象とするため                         |
| FAQ検索        | 問い合わせ前の自己解決機能は今後の課題とするため                 |
| 添付ファイル   | スクリーンショットや証憑資料の管理には追加設計が必要なため       |
| 操作履歴       | 誰がいつ更新したかの履歴管理は今後の課題とするため               |
| 本格承認フロー | 仕様が複雑化するため、Ver.2では対象外とするため                  |

## 14. まとめ

Ver.1では、問い合わせをExcel台帳で管理していた状態から、StreamlitとSQLiteを使って一元管理するフローへ変更しました。

Ver.2では、さらに要対応アラート、カテゴリ別入力フォーム、FAQ候補管理、依頼者向け確認を追加し、問い合わせ管理だけでなく、対応漏れ防止、確認往復削減、ナレッジ化、依頼者側の進捗確認までを含む業務フローに拡張しました。

本システムでは、問い合わせ対応そのものを完全に自動化するのではなく、受付、記録、確認、進捗管理、期限管理、報告作成といった管理作業を効率化することを目的とします。
