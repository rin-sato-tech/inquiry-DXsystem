# 業務フロー設計

## 1. DX化前の業務フロー

DX化前は、社員からの問い合わせが電話、口頭、メール、チャット、Excelなど複数経路で管理部に届いている。

```text
社員：電話・口頭・メール・チャット・Excelで問い合わせ
 ↓
管理部担当者が内容を確認
 ↓
必要情報が不足していれば個別に確認
 ↓
Excel台帳へ手作業で転記
 ↓
担当者を設定
 ↓
対応状況を手作業で更新、期限超過や未対応を目視で確認
 ↓
週次・月次でExcel集計
 ↓
管理部長・経営層へ報告
```

## 2. DX化前の問題点

| 工程     | 問題                                                        |
| -------- | ----------------------------------------------------------- |
| 受付     | 受付経路が分散しており、問い合わせの所在が分かりにくい      |
| 内容確認 | 初回問い合わせ時に必要情報が不足しやすい                    |
| 転記     | 電話・口頭・チャットの内容を後からExcelへ入力する必要がある |
| 進捗管理 | ステータス更新が後回しになりやすい                          |
| 期限管理 | Excelを目視確認しないと期限超過に気づきにくい               |
| 報告作成 | 週次・月次で手作業集計が必要                                |
| 属人化   | PC・システム、アカウント・権限の対応が原田に集中しやすい    |

## 3. DX化後の業務フロー

DX化後は、問い合わせをStreamlitフォームから登録し、SQLiteで一元管理する。

```text
社員または管理部担当者
 ↓
Streamlitの問い合わせ登録フォームから入力
 ↓
SQLiteに問い合わせデータを保存
 ↓
管理部が一覧画面で確認
 ↓
担当者・ステータス・対応内容を更新
 ↓
Pythonで期限超過・対応日数を自動判定
 ↓
Streamlitで未対応・期限超過・担当者別件数を確認
 ↓
Tableau用CSVを出力
 ↓
Tableauで報告用ダッシュボードを確認
```

## 4. DX化後に改善される点

| 観点     | DX化前                                    | DX化後                 |
| -------- | ----------------------------------------- | ---------------------- |
| 受付     | 電話・口頭・メール・チャット・Excelに分散 | フォーム登録に統一     |
| 記録     | Excel台帳に手入力                         | SQLiteに保存           |
| 必要情報 | 後から個別確認                            | フォーム項目で取得     |
| 進捗管理 | Excel上で手作業更新                       | Streamlitで更新        |
| 期限管理 | 目視確認                                  | 自動判定               |
| 報告作成 | Excelで手集計                             | CSV出力・Tableau可視化 |
| 属人化   | 担当者の記憶に依存                        | データとして共有       |

## 5. 原田の繁忙日の業務例

原田は、PC・システム、アカウント・権限に関する問い合わせを主に担当する。

通常日は1日1〜3件程度を担当するが、月初、月末、週次報告日、システム不具合発生日には5〜6件程度に増える。

繁忙日には、以下のような問い合わせが重なる。

| 時刻  | 問い合わせ                     | 経路     | 状態     |
| ----- | ------------------------------ | -------- | -------- |
| 9:20  | 共有フォルダ権限付与           | メール   | 完了     |
| 9:45  | PCが起動しない                 | 電話     | 対応中   |
| 10:20 | 販売管理システムのエラー       | チャット | 完了     |
| 11:30 | 複合機で印刷できない           | 口頭     | 完了     |
| 13:30 | 勤怠システムにログインできない | 電話     | 完了     |
| 14:10 | 新入社員アカウント作成         | メール   | 情報待ち |

この日は問い合わせ対応と台帳更新・週次集計に約4時間を要する。

ただし、DX化で削減できるのは、実対応そのものではなく、主に受付確認、転記、ステータス更新、期限確認、報告用集計である。
