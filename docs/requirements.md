# 社内問い合わせ管理システム 要件定義

## 1. 目的

本ドキュメントは、社内問い合わせ管理システムの要件を整理するものです。

本システムは、従業員30名規模の架空企業「株式会社西日本アーバン設備」を題材に、管理部へ寄せられる社内問い合わせ・依頼対応業務を効率化するための簡易DXアプリです。

Ver.1では、問い合わせの受付、記録、進捗管理、期限超過判定、集計、Tableau連携用CSV出力を対象としました。
Ver.2では、Ver.1の問い合わせ管理機能を拡張し、問い合わせ削減や自己解決支援につながる機能を追加しています。

## 2. 背景

株式会社西日本アーバン設備は、建材・住宅設備機器の卸売を行う従業員30名規模の企業です。

管理部は、総務、人事、経理、情報システム業務を少人数で兼務しています。

社員からの問い合わせは、以下のような複数経路で発生しています。

- 電話
- 口頭
- メール
- チャット
- Excel

DX化前は、これらの問い合わせを管理部担当者がExcel台帳へ手作業で転記していました。その結果、問い合わせの登録漏れ、ステータス更新漏れ、期限超過の見落とし、担当者別負荷の不透明さ、週次・月次報告作成の負担が発生していました。
また、初回問い合わせ時点で必要情報が不足し、管理部と依頼者の間で確認往復が発生することも課題でした。

## 3. 現状の課題

DX化前の主な課題は以下です。

| 課題                   | 内容                                                            |
| ---------------------- | --------------------------------------------------------------- |
| 受付経路の分散         | 電話、口頭、メール、チャット、Excelに問い合わせが分散している   |
| 登録漏れ               | 電話や口頭で受けた問い合わせが台帳に登録されないことがある      |
| 情報不足               | 初回問い合わせ時に必要情報が不足し、確認の往復が発生する        |
| 転記負担               | 管理部担当者がExcel台帳へ手入力している                         |
| ステータス更新漏れ     | 実際には完了していても台帳上は対応中のままになることがある      |
| 期限超過の見落とし     | Excelを目視確認しないと遅延案件に気づきにくい                   |
| 担当者未設定の見落とし | 誰が対応するか決まっていない問い合わせが埋もれる                |
| 報告作成負担           | 週次・月次報告のために手作業集計が必要                          |
| 属人化                 | PC・システム、アカウント・権限関連の対応が特定担当者に集中する  |
| ナレッジ化不足         | よくある問い合わせがFAQ化されず、同種の問い合わせが繰り返される |
| 依頼者側の進捗不明     | 依頼者が自分の問い合わせ状況を確認しにくい                      |

## 4. 対象業務

対象とする業務は以下です。

- 社内問い合わせの受付
- 問い合わせ内容の記録
- 担当者の設定
- ステータス管理
- 対応内容の記録
- 期限超過の確認
- 要対応案件の確認
- 問い合わせ件数の集計
- 作業時間の集計
- FAQ候補の管理
- 依頼者向けの問い合わせ状況確認
- Tableau連携用CSVの出力

問い合わせ対応そのものを完全に自動化するのではなく、問い合わせ管理に伴う事務作業を削減し、管理部が対応判断や実作業に集中できる状態を目指します。

## 5. 対象カテゴリ

対象カテゴリは以下です。

| カテゴリ         | 主な内容                                   |
| ---------------- | ------------------------------------------ |
| PC・システム     | PC不具合、システムエラー、周辺機器トラブル |
| アカウント・権限 | システム権限、フォルダ権限、アカウント設定 |
| 経費・請求       | 経費精算、請求書、支払関連                 |
| 勤怠・労務       | 勤怠修正、休暇、労務関連                   |
| 備品・設備       | 備品購入、設備不具合、社内設備             |
| 契約・書類       | 契約書、社内書類、申請書類                 |
| 車両・配送       | 社用車、配送、物流関連                     |
| その他           | 上記に分類しにくい問い合わせ               |

## 6. 現状工数

月間問い合わせ件数は150件とします。
DX化前の問い合わせ管理作業時間は、月約25時間と想定します。この25時間には、問い合わせ対応そのものではなく、主に以下の管理作業を含めます。

- 依頼内容の確認・整理
- Excel台帳への転記
- ステータス更新
- 日次の進捗確認
- 期限超過・未対応確認
- 担当者別負荷の確認
- 週次報告作成
- 月次集計・報告作成

## 7. 目標

DX化後は、問い合わせ管理作業を月約25時間から月約7時間へ削減することを目標とします。削減対象は、主に以下です。

- 転記作業
- 必要情報の確認作業
- ステータス確認作業
- 期限超過確認作業
- 担当者別負荷確認
- 集計・報告作成作業

想定削減効果は以下です。

| 項目                   | DX化前      | DX化後      |
| ---------------------- | ----------- | ----------- |
| 問い合わせ管理作業時間 | 約25時間/月 | 約7時間/月  |
| 削減時間               | -           | 約18時間/月 |
| 削減率                 | -           | 約70%       |

Ver.2では、上記に加えて以下を目標とします。

| 追加目標         | 内容                                                               |
| ---------------- | ------------------------------------------------------------------ |
| 対応漏れ防止     | 要対応アラートにより、期限超過や担当者未設定を見つけやすくする     |
| 確認往復削減     | カテゴリ別入力フォームにより、初回問い合わせ時点で必要情報を集める |
| 問い合わせ削減   | FAQ候補管理により、頻出問い合わせのナレッジ化につなげる            |
| 進捗確認負担削減 | 依頼者向け確認画面により、依頼者が状況を確認できるようにする       |
| 改善状況の可視化 | Ver.2追加集計とTableau出力により、改善状況を数値で確認する         |

## 8. 利用者

本システムの主な利用者は以下です。

| 利用者       | 主な利用内容                                               |
| ------------ | ---------------------------------------------------------- |
| 管理部担当者 | 問い合わせ登録、ステータス更新、対応内容記録、アラート確認 |
| 管理部責任者 | 件数、期限超過、担当者別負荷、FAQ候補、作業時間の確認      |
| 依頼者       | 自分の問い合わせ状況の確認                                 |
| 経営層       | 月次傾向、削減効果、管理部負荷の確認                       |

Ver.2の依頼者向け確認画面はデモ用であり、本格運用ではログイン認証と権限管理が必要です。

## 11. データ要件

メインテーブルは `inquiries` とします。主なデータ項目は以下です。

| 分類           | 主な項目                                                                         |
| -------------- | -------------------------------------------------------------------------------- |
| 基本情報       | 問い合わせID、受付日、受付時刻、依頼者、部署、受付経路                           |
| 問い合わせ内容 | カテゴリ、小分類、問い合わせ内容、不足情報、カテゴリ別追加情報、優先度、希望期限 |
| 対応状況       | 担当者、ステータス、対応内容、記録・管理上の問題、完了日                         |
| 工数情報       | 管理作業時間、実対応時間                                                         |
| FAQ候補        | FAQ候補フラグ、FAQタイトル、FAQ回答案                                            |
| 依頼者向け表示 | 依頼者向け表示フラグ                                                             |
| システム管理   | 作成日時、更新日時、最終ステータス変更日時                                       |

Ver.2で追加した主なカラムは以下です。

| カラム名                 | 内容                                       |
| ------------------------ | ------------------------------------------ |
| `faq_candidate`          | FAQ候補かどうか                            |
| `faq_title`              | FAQタイトル                                |
| `faq_answer`             | FAQ回答案                                  |
| `additional_info`        | カテゴリ別入力フォームで入力された追加情報 |
| `requester_visible`      | 依頼者向け画面に表示するか                 |
| `last_status_changed_at` | 最終ステータス変更日時                     |

## 14. 実装対象外

Ver.2時点で実装対象外とする機能は以下です。

| 対象外機能           | 理由                                                             |
| -------------------- | ---------------------------------------------------------------- |
| ログイン認証         | Ver.2ではデモ画面として実装し、本格認証は今後の課題とするため    |
| 権限管理             | 依頼者、管理部、管理者で表示範囲を分けるには追加設計が必要なため |
| メール通知           | ステータス変更や期限前通知は今後の拡張対象とするため             |
| Slack連携            | 社内通知ツールとの連携は今後の拡張対象とするため                 |
| FAQ公開ページ        | Ver.2ではFAQ候補管理までを対象とするため                         |
| FAQ検索              | FAQ候補を依頼者向けに検索・公開する仕組みは今後の課題とするため  |
| 添付ファイル         | スクリーンショットや証憑資料の管理には追加設計が必要なため       |
| 操作履歴             | 誰がいつ更新したかの履歴管理は今後の課題とするため               |
| AI自動分類           | データ量が少ない段階では対象外とするため                         |
| 本格承認ワークフロー | 仕様が複雑化するためVer.2では対象外とするため                    |
