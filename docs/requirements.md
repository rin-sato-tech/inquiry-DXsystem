# 社内問い合わせ管理システム 要件定義

## 1. 背景

株式会社西日本アーバン設備は、従業員30名の建材・住宅設備機器の卸売会社である。

管理部4名が、総務、人事、経理、情報システム業務を兼務している。社員からの問い合わせは、電話、口頭、メール、チャット、Excelなど複数経路で発生しており、管理部担当者が内容をExcel台帳へ手作業で転記している。

そのため、登録漏れ、ステータス更新漏れ、期限超過の見落とし、担当者別負荷の不透明さ、週次・月次報告作成の負担が発生している。

## 2. 目的

本システムの目的は、管理部に寄せられる社内問い合わせ・依頼対応業務について、受付、記録、進捗管理、期限確認、報告作成を効率化することである。

問い合わせ対応そのものを完全に自動化するのではなく、問い合わせ管理に伴う事務作業を削減し、管理部が対応判断や実作業に集中できる状態を目指す。

## 3. 対象業務

対象とする業務は以下である。

- 社内問い合わせの受付
- 問い合わせ内容の記録
- 担当者の設定
- ステータス管理
- 対応内容の記録
- 期限超過の確認
- 問い合わせ件数の集計
- Tableau連携用CSVの出力

## 4. 対象カテゴリ

対象カテゴリは以下である。

- PC・システム
- アカウント・権限
- 経費・請求
- 勤怠・労務
- 備品・設備
- 契約・書類
- 車両・配送
- その他

## 5. 現状の課題

DX化前の主な課題は以下である。

| 課題               | 内容                                                          |
| ------------------ | ------------------------------------------------------------- |
| 受付経路の分散     | 電話、口頭、メール、チャット、Excelに問い合わせが分散している |
| 登録漏れ           | 電話や口頭で受けた問い合わせが台帳に登録されないことがある    |
| 情報不足           | 初回問い合わせ時に必要情報が不足し、確認の往復が発生する      |
| 転記負担           | 管理部担当者がExcel台帳へ手入力している                       |
| ステータス更新漏れ | 実際には完了していても台帳上は対応中のままになることがある    |
| 期限超過の見落とし | Excelを目視確認しないと遅延案件に気づきにくい                 |
| 報告作成負担       | 週次・月次報告のために手作業集計が必要                        |
| 属人化             | PC・システム、アカウント・権限の対応が原田に集中している      |

## 6. 現状工数

月間問い合わせ件数は150件とする。

問い合わせ管理にかかる作業時間は月約25時間とする。

この25時間には、問い合わせ対応そのものではなく、主に以下の管理作業を含める。

- 依頼内容の確認・整理
- Excel台帳への転記
- ステータス更新
- 日次の進捗確認
- 期限超過・未対応確認
- 週次報告作成
- 月次集計・報告作成

## 7. 目標

DX化後は、問い合わせ管理作業を月約25時間から月約7時間へ削減する。

削減対象は、主に以下である。

- 転記作業
- 必要情報の確認作業
- ステータス確認作業
- 期限超過確認作業
- 集計・報告作成作業

想定削減効果は、月約18時間、削減率約70%とする。

## 8. 実装範囲

今回実装する機能は以下である。

| 機能               | 内容                                           |
| ------------------ | ---------------------------------------------- |
| 問い合わせ一覧表示 | SQLiteに保存された問い合わせを一覧表示する     |
| 検索・絞り込み     | 部署、カテゴリ、担当者、ステータスで絞り込む   |
| 新規登録           | Streamlitフォームから問い合わせを登録する      |
| ステータス更新     | 担当者、ステータス、対応内容、完了日を更新する |
| 期限超過判定       | 未完了かつ期限切れの案件を自動抽出する         |
| 簡易集計           | カテゴリ別、担当者別、ステータス別に集計する   |
| CSV出力            | Tableau用CSVを出力する                         |

## 9. 実装対象外

今回実装しない機能は以下である。

| 対象外機能           | 理由                                 |
| -------------------- | ------------------------------------ |
| ログイン認証         | 初期版では実装負荷が高いため         |
| 権限管理             | 管理部内利用のデモとしては不要なため |
| Slack連携            | 初期版では本質から外れるため         |
| メール通知           | 初期版では不要なため                 |
| FAQ検索              | 発展課題とするため                   |
| AI自動分類           | データ量が少ない段階では不要なため   |
| 本格承認ワークフロー | 仕様が複雑化するため                 |

## 10. 技術構成

| 技術      | 用途                            |
| --------- | ------------------------------- |
| Python    | データ処理、集計、期限超過判定  |
| pandas    | CSV読み込み、集計、加工         |
| SQLite    | 問い合わせデータの保存          |
| Streamlit | 管理部向け操作画面              |
| Tableau   | 管理者向けダッシュボード        |
| CSV       | Tableau連携、バックアップ、出力 |

## 11. 成功条件

以下を満たせばMVP完成とする。

- ダミーCSVをSQLiteに取り込める
- Streamlitで問い合わせ一覧を表示できる
- 部署、カテゴリ、担当者、ステータスで絞り込める
- 新規問い合わせを登録できる
- ステータス、担当者、対応内容を更新できる
- 期限超過案件を抽出できる
- カテゴリ別、担当者別、ステータス別に集計できる
- Tableau用CSVを出力できる
