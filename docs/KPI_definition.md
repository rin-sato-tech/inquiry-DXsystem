# KPI定義書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システムで使用するKPIを定義するものです。

本システムでは、問い合わせ管理業務の状況を把握し、管理部の負担、対応状況、期限超過、FAQ候補、追加情報入力状況などを可視化します。

Ver.1では、問い合わせ件数、未完了件数、期限超過件数、対応日数、作業時間などを中心にKPIを定義しました。Ver.2では、以下の機能追加に伴い、KPIを拡張しています。

- 要対応アラート
- FAQ候補管理
- カテゴリ別入力フォーム
- 依頼者向け確認画面
- Ver.2追加集計
- Tableau出力列の拡張

## 2. KPI設計の方針

KPIは、以下の観点で整理します。

| 観点           | 内容                                             |
| -------------- | ------------------------------------------------ |
| 業務量         | 問い合わせ件数、カテゴリ別件数、部署別件数など   |
| 滞留状況       | 未完了件数、ステータス別件数、期限超過件数など   |
| 対応スピード   | 平均対応日数、カテゴリ別平均対応日数など         |
| 作業負荷       | 管理作業時間、実対応時間、担当者別作業時間など   |
| 対応漏れリスク | 要対応アラート、担当者未設定、情報待ち長期化など |
| ナレッジ化     | FAQ候補件数、カテゴリ別FAQ候補件数など           |
| 入力品質       | 追加情報入力件数、追加情報入力率など             |
| 依頼者向け表示 | 依頼者向け表示件数、非表示件数など               |
| 効果測定       | 削減時間、削減率、Tableauでの可視化項目など      |

## 3. 基本KPI

### 3.1 問い合わせ件数

| 項目     | 内容                             |
| -------- | -------------------------------- |
| KPI      | 問い合わせ件数                   |
| 定義     | 登録された問い合わせの件数       |
| 計算方法 | `COUNT(request_id)`              |
| 目的     | 問い合わせ全体の業務量を把握する |

### 3.2 未完了件数

| 項目     | 内容                                 |
| -------- | ------------------------------------ |
| KPI      | 未完了件数                           |
| 定義     | ステータスが完了以外の問い合わせ件数 |
| 計算方法 | `status != "完了"` の件数            |
| 目的     | 滞留している問い合わせを把握する     |

### 3.3 完了件数

| 項目     | 内容                             |
| -------- | -------------------------------- |
| KPI      | 完了件数                         |
| 定義     | ステータスが完了の問い合わせ件数 |
| 計算方法 | `status == "完了"` の件数        |
| 目的     | 処理済み問い合わせ件数を把握する |

### 3.4 期限超過件数

| 項目     | 内容                                           |
| -------- | ---------------------------------------------- |
| KPI      | 期限超過件数                                   |
| 定義     | 未完了、かつ希望期限を過ぎている問い合わせ件数 |
| 計算方法 | `status != "完了" and due_date < 今日`         |
| 目的     | 遅延リスクを把握する                           |

### 3.5 平均対応日数

| 項目     | 内容                                               |
| -------- | -------------------------------------------------- |
| KPI      | 平均対応日数                                       |
| 定義     | 完了済み問い合わせの受付日から完了日までの平均日数 |
| 計算方法 | `AVG(completed_date - request_date)`               |
| 目的     | 対応スピードを把握する                             |

### 3.6 管理作業時間

| 項目     | 内容                                                       |
| -------- | ---------------------------------------------------------- |
| KPI      | 管理作業時間                                               |
| 定義     | 転記、確認、進捗管理、期限確認、報告作成などにかかった時間 |
| 計算方法 | `SUM(management_minutes) / 60`                             |
| 目的     | 問い合わせ管理にかかる事務負担を把握する                   |

### 3.7 実対応時間

| 項目     | 内容                                               |
| -------- | -------------------------------------------------- |
| KPI      | 実対応時間                                         |
| 定義     | PC確認、権限付与、書類確認など実対応にかかった時間 |
| 計算方法 | `SUM(actual_response_minutes) / 60`                |
| 目的     | 実作業負荷を把握する                               |

### 3.8 問い合わせ関連時間

| 項目     | 内容                                                            |
| -------- | --------------------------------------------------------------- |
| KPI      | 問い合わせ関連時間                                              |
| 定義     | 管理作業時間と実対応時間の合計                                  |
| 計算方法 | `(SUM(management_minutes) + SUM(actual_response_minutes)) / 60` |
| 目的     | 問い合わせ対応に関わる総負荷を把握する                          |

## 4. 分類別KPI

| KPI                | 定義                               | 目的                                   |
| ------------------ | ---------------------------------- | -------------------------------------- |
| カテゴリ別件数     | `category` ごとの問い合わせ件数    | 問い合わせが多い領域を把握する         |
| サブカテゴリ別件数 | `subcategory` ごとの問い合わせ件数 | 具体的な頻出内容を把握する             |
| 部署別件数         | `department` ごとの問い合わせ件数  | どの部署から問い合わせが多いか把握する |
| 担当者別件数       | `assignee` ごとの問い合わせ件数    | 管理部内の負荷偏りを把握する           |
| ステータス別件数   | `status` ごとの問い合わせ件数      | 対応状況を把握する                     |
| 受付経路別件数     | `channel` ごとの問い合わせ件数     | 受付経路の分散状況を把握する           |
| 優先度別件数       | `priority` ごとの問い合わせ件数    | 緊急度の傾向を把握する                 |

## 5. 作業時間KPI

| KPI                    | 定義                                          | 目的                                         |
| ---------------------- | --------------------------------------------- | -------------------------------------------- |
| 担当者別管理作業時間   | 担当者ごとの `management_minutes` 合計        | 管理作業負荷の偏りを把握する                 |
| 担当者別実対応時間     | 担当者ごとの `actual_response_minutes` 合計   | 実対応負荷の偏りを把握する                   |
| 担当者別合計作業時間   | 担当者ごとの管理作業時間と実対応時間の合計    | 担当者別の総負荷を把握する                   |
| カテゴリ別管理作業時間 | カテゴリごとの `management_minutes` 合計      | 管理負荷が大きいカテゴリを把握する           |
| カテゴリ別実対応時間   | カテゴリごとの `actual_response_minutes` 合計 | 実対応負荷が大きいカテゴリを把握する         |
| カテゴリ別合計作業時間 | カテゴリごとの管理作業時間と実対応時間の合計  | 問い合わせ対応負荷が大きいカテゴリを把握する |

## 6. Ver.2追加KPI

### 6.1 要対応アラートKPI

Ver.2では、対応漏れや期限超過を防ぐため、要対応アラートに関するKPIを追加します。

| KPI                  | 定義                                     | 目的                                         |
| -------------------- | ---------------------------------------- | -------------------------------------------- |
| 要対応アラート件数   | 何らかのアラートに該当する問い合わせ件数 | 優先確認が必要な問い合わせ数を把握する       |
| 期限超過アラート件数 | 未完了、かつ希望期限を過ぎている件数     | 遅延案件を把握する                           |
| 本日期限件数         | 未完了、かつ希望期限が今日の件数         | 当日中に確認すべき問い合わせを把握する       |
| 期限間近件数         | 未完了、かつ希望期限が明日までの件数     | 期限が近い問い合わせを把握する               |
| 担当者未設定件数     | 担当者が空欄の問い合わせ件数             | 対応主体が決まっていない問い合わせを把握する |
| 情報待ち長期化件数   | 情報待ち状態が一定日数以上続いている件数 | 依頼者確認が滞っている問い合わせを把握する   |
| アラート種別別件数   | `alert_type` ごとの件数                  | どの種類のリスクが多いか把握する             |

### 6.2 FAQ候補KPI

FAQ候補管理により、よくある問い合わせのナレッジ化状況を把握します。

| KPI                   | 定義                                  | 目的                                                  |
| --------------------- | ------------------------------------- | ----------------------------------------------------- |
| FAQ候補件数           | `faq_candidate == 1` の問い合わせ件数 | FAQ化できそうな問い合わせの蓄積状況を把握する         |
| カテゴリ別FAQ候補件数 | FAQ候補をカテゴリ別に集計した件数     | FAQ化すべき領域を把握する                             |
| FAQ候補率             | FAQ候補件数 ÷ 完了件数                | 完了済み問い合わせのうちFAQ化候補になる割合を把握する |
| FAQ候補CSV出力件数    | FAQ候補CSVに出力される件数            | FAQ候補データの出力対象を把握する                     |

### 6.3 カテゴリ別追加情報KPI

カテゴリ別入力フォームにより、初回問い合わせ時点で必要情報をどの程度集められているかを把握します。

| KPI                      | 定義                                       | 目的                                                 |
| ------------------------ | ------------------------------------------ | ---------------------------------------------------- |
| 追加情報入力件数         | `additional_info` が空でない問い合わせ件数 | カテゴリ別追加情報が入力された件数を把握する         |
| 追加情報未入力件数       | `additional_info` が空の問い合わせ件数     | 追加情報が不足している問い合わせを把握する           |
| 追加情報入力率           | 追加情報入力件数 ÷ 問い合わせ件数          | 初回入力時点で必要情報を集められている割合を把握する |
| カテゴリ別追加情報入力率 | カテゴリごとの追加情報入力率               | どのカテゴリで入力不足が多いか把握する               |

### 6.4 依頼者向け表示KPI

依頼者向け確認画面で表示される問い合わせ件数を把握します。

| KPI                  | 定義                                      | 目的                                             |
| -------------------- | ----------------------------------------- | ------------------------------------------------ |
| 依頼者向け表示件数   | `requester_visible == 1` の問い合わせ件数 | 依頼者向け画面に表示される件数を把握する         |
| 依頼者向け非表示件数 | `requester_visible == 0` の問い合わせ件数 | 依頼者向け画面に出さない問い合わせ件数を把握する |
| 依頼者向け表示率     | 依頼者向け表示件数 ÷ 問い合わせ件数       | 依頼者向けに公開可能な問い合わせの割合を把握する |

## 9. 効率化効果の定義

DX化前の問い合わせ管理作業時間を月約25時間とします。DX化後の問い合わせ管理作業時間を月約7時間とします。削減時間は以下で計算します。

```text
25時間 - 7時間 = 18時間
```

削減率は以下で計算します。

```text
18時間 ÷ 25時間 = 72%
```

ポートフォリオ上では、丸めて以下のように表現します。

```text
問い合わせ管理作業を月約25時間から月約7時間へ削減し、
月約18時間、約70%の効率化を見込む。
```

この削減効果は、問い合わせ対応そのものを70%削減するという意味ではありません。削減対象は、以下の管理作業です。

```text
問い合わせ内容の転記
必要情報の確認
ステータス更新
進捗確認
期限超過確認
担当者未設定確認
週次・月次報告作成
```

PC確認、権限付与、パスワード再設定、契約書確認などの実対応は引き続き必要です。

## 12. KPI利用上の注意点

KPIは、業務改善の状況を把握するための目安です。数値だけで評価すると、以下のような誤解が起こる可能性があります。

| 注意点               | 内容                                                                                  |
| -------------------- | ------------------------------------------------------------------------------------- |
| 問い合わせ件数の減少 | FAQや自己解決が進んだ結果か、単に記録されなくなっただけかを確認する必要がある         |
| 期限超過件数の減少   | 実際に改善したのか、期限設定が甘くなっただけかを確認する必要がある                    |
| FAQ候補件数の増加    | FAQ化の準備が進んだ一方で、公開・検索まで進んでいなければ問い合わせ削減には直結しない |
| 追加情報入力率の上昇 | 入力率が高くても、入力内容の質が低ければ確認往復は減らない                            |
| 依頼者向け表示件数   | 表示件数が多くても、ログイン認証・権限管理がない段階では本格運用には不足がある        |
| 作業時間の減少       | 実対応の質が落ちていないかを確認する必要がある                                        |

## 13. まとめ

Ver.1では、問い合わせ件数、未完了件数、期限超過件数、対応日数、作業時間などを中心にKPIを定義しました。
Ver.2では、要対応アラート、FAQ候補、カテゴリ別追加情報、依頼者向け表示に関するKPIを追加しています。

これにより、単に問い合わせを記録・管理するだけでなく、対応漏れ防止、確認往復削減、ナレッジ化、依頼者側の進捗確認まで含めて改善状況を把握できるようにしています。
