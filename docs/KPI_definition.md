# KPI定義書

## 1. 目的

本システムでは、問い合わせ管理業務の状況を把握し、管理部の負担や滞留状況を可視化するためにKPIを定義する。

## 2. 基本KPI

| KPI            | 定義                                 | 目的                           |
| -------------- | ------------------------------------ | ------------------------------ |
| 問い合わせ件数 | 登録された問い合わせの件数           | 業務量の把握                   |
| 未完了件数     | statusが完了以外の件数               | 滞留状況の把握                 |
| 完了件数       | statusが完了の件数                   | 処理済み件数の把握             |
| 期限超過件数   | 未完了かつdue_dateが今日より前の件数 | 遅延リスクの把握               |
| 平均対応日数   | 完了案件のresponse_days平均          | 対応スピードの把握             |
| 管理作業時間   | management_minutesの合計             | 転記・確認・報告作成負担の把握 |
| 実対応時間     | actual_response_minutesの合計        | 実作業負荷の把握               |

## 3. 分類別KPI

| KPI                | 定義                  | 目的                               |
| ------------------ | --------------------- | ---------------------------------- |
| カテゴリ別件数     | categoryごとの件数    | 問い合わせが多い領域を把握         |
| サブカテゴリ別件数 | subcategoryごとの件数 | 具体的な頻出内容を把握             |
| 部署別件数         | departmentごとの件数  | どの部署から問い合わせが多いか把握 |
| 担当者別件数       | assigneeごとの件数    | 管理部内の負荷偏りを把握           |
| ステータス別件数   | statusごとの件数      | 対応状況を把握                     |
| 受付経路別件数     | channelごとの件数     | DX化前の分散状況を把握             |
| 優先度別件数       | priorityごとの件数    | 緊急度の傾向を把握                 |

## 4. Tableauで可視化する項目

Tableauでは以下を可視化する。

| ビュー               | 内容                                                        |
| -------------------- | ----------------------------------------------------------- |
| 受付経路別件数       | 電話・口頭・メール・チャット・Excelに分散していた状況を示す |
| カテゴリ別件数       | 問い合わせの多いカテゴリを示す                              |
| 担当者別件数         | 原田に問い合わせが集中している状況を示す                    |
| ステータス別件数     | 未対応・対応中・完了の状態を示す                            |
| 期限超過件数         | 遅延案件の有無を示す                                        |
| 管理作業時間         | 問い合わせ管理にかかる時間を示す                            |
| 部署別問い合わせ件数 | どの部署から問い合わせが多いかを示す                        |

## 5. 効率化効果の定義

DX化前の問い合わせ管理作業時間を月約25時間とする。

DX化後の問い合わせ管理作業時間を月約7時間とする。

削減時間は以下で計算する。

```text
25時間 - 7時間 = 18時間

削減率は以下で計算する。

18時間 ÷ 25時間 = 72%
```

ポートフォリオ上では、丸めて以下のように表現する。

```text
問い合わせ管理作業を月約25時間から月約7時間へ削減し、月約18時間、約70%の効率化を見込む。
```

## 6. 注意点

この削減効果は、問い合わせ対応そのものを70%削減するという意味ではない。

削減対象は、以下の管理作業である。

```text
問い合わせ内容の転記
必要情報の確認
ステータス更新
進捗確認
期限超過確認
週次・月次報告作成
```

PC確認、権限付与、パスワード再設定、契約書確認などの実対応は引き続き必要である。
