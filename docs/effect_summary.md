# 効果説明

## 1. 目的

本ドキュメントは、社内問い合わせ管理システムを導入した場合に見込まれる効果を整理するものです。

本システムは、問い合わせ対応そのものを完全に自動化するものではありません。主な目的は、問い合わせ管理に伴う以下の作業を効率化することです。

- 問い合わせ内容の記録
- Excel台帳への転記
- 必要情報の確認
- ステータス確認
- 期限超過確認
- 担当者別負荷確認
- 週次・月次報告用の集計
- Tableau用CSV出力

Ver.2では、上記に加えて、要対応アラート、FAQ候補管理、カテゴリ別入力フォーム、依頼者向け確認画面を追加し、対応漏れ防止、確認往復削減、問い合わせ削減への布石も効果対象に含めます。

## 2. 対象業務

対象業務は、管理部に寄せられる社内問い合わせ・依頼対応業務です。
対象企業は、従業員30名規模の架空企業「株式会社西日本アーバン設備」とします。
管理部は、総務、人事、経理、情報システム業務を少人数で兼務しており、以下のような問い合わせを受け付けます。

| カテゴリ         | 主な内容                                   |
| ---------------- | ------------------------------------------ |
| PC・システム     | PC不具合、システムエラー、周辺機器トラブル |
| アカウント・権限 | システム権限、フォルダ権限、アカウント設定 |
| 経費・請求       | 経費精算、請求書、支払関連                 |
| 勤怠・労務       | 勤怠修正、休暇、労務関連                   |
| 備品・設備       | 備品購入、設備不具合、社内設備             |
| 契約・書類       | 契約書、社内書類、申請書類                 |
| 車両・配送       | 社用車、配送、物流関連                     |
| その他           | 上記に分類しにくい問い合わせ               |

## 3. DX化前の状態

DX化前は、問い合わせが以下の複数経路に分散していました。

- 電話
- 口頭
- メール
- チャット
- Excel

管理部担当者は、これらの問い合わせを手作業でExcel台帳に転記していました。また、ステータス、担当者、期限超過、未対応案件、担当者別件数などもExcelを目視で確認していました。
そのため、以下の問題が発生していました。

| 問題                   | 内容                                                     |
| ---------------------- | -------------------------------------------------------- |
| 登録漏れ               | 電話・口頭の問い合わせが台帳に残らないことがある         |
| 転記負担               | 問い合わせ内容をExcelへ手入力する必要がある              |
| 情報不足               | 初回問い合わせ時点で必要情報が不足し、確認往復が発生する |
| 更新漏れ               | 対応済みでもステータスが更新されないことがある           |
| 期限超過の見落とし     | 期限切れ案件を目視で確認する必要がある                   |
| 担当者未設定の見落とし | 誰が対応するか決まっていない問い合わせが埋もれる         |
| 報告作成負担           | 週次・月次報告のために手作業集計が必要                   |
| 属人化                 | よくある問い合わせや対応方法が担当者の記憶に残りやすい   |
| 依頼者側の進捗不明     | 依頼者が管理部へ個別に状況確認する必要がある             |

## 4. Ver.1導入後の状態

Ver.1では、問い合わせをSQLiteで一元管理し、Streamlit上で一覧確認、登録、更新、集計を行えるようにしました。
また、Tableau用CSVを出力することで、問い合わせ件数、未完了件数、期限超過件数、担当者別負荷などを可視化できるようにしました。
Ver.1で改善される主な点は以下です。

| 観点           | DX化前                      | Ver.1導入後                                      |
| -------------- | --------------------------- | ------------------------------------------------ |
| 記録           | Excel台帳へ手入力           | フォーム・DB登録                                 |
| 一覧確認       | Excelを目視確認             | Streamlitで一覧確認                              |
| 絞り込み       | Excelフィルタを手作業で利用 | 部署、カテゴリ、担当者、ステータスなどで絞り込み |
| 期限超過       | 目視確認                    | 自動判定                                         |
| ステータス更新 | Excelを直接編集             | 更新画面から入力                                 |
| 集計           | Excelで手作業集計           | Pythonで自動集計                                 |
| 報告           | Excel集計をもとに資料作成   | Tableau用CSVを出力                               |

## 5. Ver.2導入後の状態

Ver.2では、Ver.1の問い合わせ管理機能に加えて、以下の機能を追加しました。

- 要対応アラート
- FAQ候補管理
- カテゴリ別入力フォーム
- 依頼者向け確認画面
- Ver.2追加集計
- Tableau出力列の拡張

これにより、単に問い合わせを記録・管理するだけでなく、対応漏れ防止、確認往復削減、FAQ化の準備、依頼者側の進捗確認まで支援できるようになります。

| 観点        | Ver.1            | Ver.2                                                          |
| ----------- | ---------------- | -------------------------------------------------------------- |
| 期限管理    | 期限超過を判定   | 本日期限、期限間近、担当者未設定、情報待ち長期化もアラート表示 |
| 入力情報    | 基本項目を入力   | カテゴリ別追加項目で必要情報も収集                             |
| ナレッジ化  | 対応内容を記録   | 完了済み問い合わせをFAQ候補として蓄積                          |
| 依頼者確認  | 管理部側で確認   | 依頼者向け確認画面で状況確認                                   |
| 集計        | 基本KPIを集計    | アラート、FAQ候補、追加情報入力率も集計                        |
| Tableau出力 | 基本集計列を出力 | Ver.2追加列も出力                                              |

## 6. 想定削減効果

月間問い合わせ件数は150件とします。DX化前の問い合わせ管理作業時間は、月約25時間と想定します。システム導入後は、管理作業時間を月約7時間まで削減することを想定します。

| 項目                   |      DX化前 |      DX化後 |
| ---------------------- | ----------: | ----------: |
| 問い合わせ管理作業時間 | 約25時間/月 |  約7時間/月 |
| 削減時間               |           - | 約18時間/月 |
| 削減率                 |           - |       約70% |

この削減効果は、問い合わせ対応そのものではなく、管理作業を対象としたものです。

## 7. 削減される作業

削減される主な作業は以下です。

| 作業               | DX化前               | DX化後                                   |
| ------------------ | -------------------- | ---------------------------------------- |
| 問い合わせ記録     | Excelへ手入力        | フォーム・DB登録                         |
| 必要情報の確認     | 不足情報を個別に確認 | カテゴリ別入力フォームで初回入力時に収集 |
| ステータス確認     | Excelを目視確認      | 一覧・フィルタで確認                     |
| 期限超過確認       | Excelを目視確認      | 自動判定・要対応アラートで確認           |
| 担当者未設定確認   | Excelを目視確認      | 担当者未設定アラートで確認               |
| 担当者別負荷確認   | 手作業集計           | 自動集計                                 |
| FAQ候補整理        | 担当者の記憶に依存   | 完了済み問い合わせからFAQ候補登録        |
| 依頼者への進捗回答 | 個別問い合わせに回答 | 依頼者向け確認画面で一部自己確認         |
| 報告作成           | Excel集計            | Tableau用CSV出力                         |

## 8. Ver.2で見込まれる追加効果

### 8.1 要対応アラートによる対応漏れ防止

Ver.2では、期限超過だけでなく、本日期限、期限間近、担当者未設定、情報待ち長期化も確認できます。これにより、管理部担当者は、優先的に対応すべき問い合わせを見つけやすくなります。

| 効果                   | 内容                                           |
| ---------------------- | ---------------------------------------------- |
| 期限超過の見落とし削減 | 期限切れ案件をアラートで確認できる             |
| 当日対応案件の把握     | 本日期限の問い合わせを確認できる               |
| 担当者未設定の防止     | 担当者が空欄の問い合わせを見つけられる         |
| 情報待ちの放置防止     | 情報待ちが長期化している問い合わせを確認できる |

### 8.2 カテゴリ別入力フォームによる確認往復削減

Ver.2では、問い合わせカテゴリに応じて追加項目を表示します。たとえば、PC・システムであれば、PC管理番号、発生時刻、エラー内容、再起動有無などを入力できます。

| 効果                         | 内容                                               |
| ---------------------------- | -------------------------------------------------- |
| 必要情報の不足を減らす       | 初回問い合わせ時点で確認すべき情報を集めやすくなる |
| 管理部からの追加確認を減らす | 不足情報の個別確認を減らせる                       |
| 対応開始を早める             | 必要情報が揃っていれば、担当者がすぐ判断しやすい   |
| FAQ候補化しやすくなる        | 問い合わせ背景や条件が残るため、後から整理しやすい |

### 8.3 FAQ候補管理による問い合わせ削減への布石

Ver.2では、完了済み問い合わせからFAQ候補を登録できます。
FAQ公開ページまでは実装していませんが、よくある問い合わせを整理し、将来的な自己解決支援につなげることができます。

| 効果                   | 内容                                             |
| ---------------------- | ------------------------------------------------ |
| 頻出問い合わせの把握   | FAQ化できそうな問い合わせを蓄積できる            |
| 対応内容の再利用       | 過去の対応内容をFAQ回答案として整理できる        |
| 属人化の緩和           | 担当者の記憶に依存していた対応を共有しやすくする |
| 問い合わせ削減への準備 | FAQ公開・検索機能への発展につなげられる          |

### 8.4 依頼者向け確認による進捗確認負担の削減

Ver.2では、問い合わせIDまたは依頼者名で問い合わせ状況を確認できる画面を追加しています。これにより、依頼者が管理部へ個別に状況確認する回数を減らすことを想定します。

| 効果                       | 内容                                         |
| -------------------------- | -------------------------------------------- |
| 依頼者の不安軽減           | 自分の問い合わせ状況を確認できる             |
| 管理部への確認連絡削減     | 「今どうなっていますか」という確認を減らせる |
| ステータス更新の重要性向上 | ステータスを更新する意味が明確になる         |
| 本格運用への布石           | 将来的なログイン認証・権限管理へ発展できる   |

## 9. 効果を確認するKPI

効果確認に使う主なKPIは以下です。

| KPI                | 確認したいこと                           |
| ------------------ | ---------------------------------------- |
| 問い合わせ件数     | 問い合わせ全体の業務量                   |
| 未完了件数         | 滞留している問い合わせ数                 |
| 期限超過件数       | 遅延している問い合わせ数                 |
| 要対応アラート件数 | 優先確認が必要な問い合わせ数             |
| 担当者未設定件数   | 対応主体が決まっていない問い合わせ数     |
| 情報待ち長期化件数 | 依頼者確認が滞っている問い合わせ数       |
| 管理作業時間       | 管理作業の負荷                           |
| 実対応時間         | 実作業の負荷                             |
| FAQ候補件数        | ナレッジ化できそうな問い合わせの蓄積状況 |
| 追加情報入力率     | カテゴリ別入力フォームの活用状況         |
| 依頼者向け表示件数 | 依頼者が確認できる問い合わせ件数         |

## 10. 効果の見せ方

ポートフォリオ上では、以下の流れで効果を説明します。

```text
DX化前
  問い合わせが複数経路に分散し、Excel台帳への転記・目視確認・手作業集計が発生していた。

Ver.1
  問い合わせをDBで一元管理し、一覧表示、登録、更新、期限超過判定、集計、Tableau出力を実装した。

Ver.2
  要対応アラート、FAQ候補管理、カテゴリ別入力フォーム、依頼者向け確認を追加し、対応漏れ防止・確認往復削減・ナレッジ化まで拡張した。

効果
  問い合わせ管理作業を月約25時間から月約7時間へ削減し、月約18時間、約70%の効率化を見込む。
```

このように説明すると、単なるアプリ作成ではなく、業務課題、改善策、効果試算を一貫して示せます。

## 11. 注意点

この削減率は、問い合わせ対応そのものが70%減るという意味ではありません。PC確認、権限付与、パスワード再設定、契約書確認、請求内容確認などの実対応は引き続き必要です。
削減対象は、主に以下の管理作業です。

- 問い合わせ内容の転記
- 必要情報の確認
- ステータス確認
- 期限超過確認
- 担当者未設定確認
- 報告用集計
- FAQ候補整理
- 進捗確認への個別回答

また、Ver.2のFAQ候補管理や依頼者向け確認画面は、本格運用に向けた前段階です。FAQ公開ページ、FAQ検索、ログイン認証、権限管理、メール通知などは今後の発展課題です。

## 12. 業務改善上の意味

本システムにより、管理部は以下を把握しやすくなります。

- どのカテゴリの問い合わせが多いか
- どの部署から問い合わせが多いか
- どの担当者に負荷が集中しているか
- どの問い合わせが期限超過しているか
- どの問い合わせが優先確認対象か
- どのカテゴリで追加情報が入力されているか
- どの問い合わせがFAQ候補になり得るか
- 月次報告に必要な数値がどこにあるか

これにより、単なる問い合わせ受付ではなく、管理部業務の改善、属人化の把握、問い合わせ削減、FAQ化、報告作成の効率化につなげることができます。

## 13. まとめ

Ver.1では、問い合わせをExcel台帳で管理していた状態から、StreamlitとSQLiteを使って一元管理する仕組みを作成しました。
Ver.2では、要対応アラート、カテゴリ別入力フォーム、FAQ候補管理、依頼者向け確認画面を追加しました。

これにより、問い合わせを記録・管理するだけでなく、対応漏れ防止、確認往復削減、ナレッジ化、依頼者側の進捗確認まで含めた改善効果を示せるようにしています。想定効果としては、問い合わせ管理作業を月約25時間から月約7時間へ削減し、月約18時間、約70%の効率化を見込んでいます。
