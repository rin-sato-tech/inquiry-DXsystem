# 効果説明

## 1. 対象業務

本システムは、管理部に寄せられる社内問い合わせ・依頼対応業務を対象とする。
対象とするのは、問い合わせ対応そのものではなく、主に以下の管理作業である。

- 問い合わせ内容の記録
- Excel台帳への転記
- ステータス更新
- 期限超過確認
- 担当者別負荷確認
- 週次・月次報告用の集計
- Tableau用CSV出力

## 2. DX化前の状態

DX化前は、問い合わせが電話、口頭、メール、チャット、Excelに分散していた。
管理部担当者は、それらの問い合わせを手作業でExcel台帳に転記し、ステータスや期限を目視で確認していた。そのため、登録漏れ、更新漏れ、期限超過の見落とし、報告作成負担が発生していた。

## 3. DX化後の状態

DX化後は、問い合わせをSQLiteで一元管理し、Streamlit上で一覧確認、登録、更新、集計を行う。
また、Tableau用CSVを出力することで、問い合わせ件数、未完了件数、期限超過件数、担当者別負荷などを可視化できる。

## 4. 想定削減効果

| 項目                   |      DX化前 |      DX化後 |
| ---------------------- | ----------: | ----------: |
| 問い合わせ管理作業時間 | 約25時間/月 |  約7時間/月 |
| 削減時間               |           - | 約18時間/月 |
| 削減率                 |           - |       約70% |

## 5. 削減される作業

| 作業             | DX化前          | DX化後               |
| ---------------- | --------------- | -------------------- |
| 問い合わせ記録   | Excelへ手入力   | フォーム・DB登録     |
| ステータス確認   | Excelを目視確認 | 一覧・フィルタで確認 |
| 期限超過確認     | 目視確認        | 自動判定             |
| 担当者別負荷確認 | 手作業集計      | 自動集計             |
| 報告作成         | Excel集計       | Tableau用CSV出力     |

## 6. 注意点

この削減率は、問い合わせ対応そのものが70%減るという意味ではない。PC確認、権限付与、パスワード再設定、契約書確認などの実対応は引き続き必要である。
削減対象は、問い合わせ管理に伴う転記、確認、集計、報告作成などの事務作業である。

## 7. 業務改善上の意味

このシステムにより、管理部は以下を把握しやすくなる。

- どのカテゴリの問い合わせが多いか
- どの部署から問い合わせが多いか
- どの担当者に負荷が集中しているか
- どの問い合わせが期限超過しているか
- 月次報告に必要な数値がどこにあるか

これにより、単なる問い合わせ受付ではなく、管理部業務の改善、属人化の把握、FAQ化候補の発見につなげることができる。
