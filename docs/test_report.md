# テスト結果報告書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システムのテスト結果を報告するものです。

Ver.1では、CSV取込、DB保存、問い合わせ一覧表示、新規登録、ステータス更新、期限超過判定、集計、Tableau用CSV出力が正常に動作することを確認しました。
Ver.2では、Ver.1の基本機能に加えて、以下の追加機能が正常に動作することを確認しました。

- 要対応アラート
- FAQ候補管理
- カテゴリ別入力フォーム
- 依頼者向け確認画面
- Ver.2追加集計
- Tableau出力列の拡張
- DB移行処理
- Ver.2スモークテスト

## 2. テスト方針

本システムでは、以下の3種類の確認を行います。

| 種別       | 内容                                                             |
| ---------- | ---------------------------------------------------------------- |
| 自動テスト | 構文確認、DB確認、スモークテスト、Tableau CSV出力確認を行う      |
| 手動テスト | Streamlit画面を操作し、登録・更新・検索・集計・CSV出力を確認する |
| 回帰確認   | Ver.2追加機能によってVer.1基本機能が壊れていないことを確認する   |

## 10. 修正した不具合・改善点

|  No | 内容                                                       | 対応                                                                             |
| --: | ---------------------------------------------------------- | -------------------------------------------------------------------------------- |
|   1 | ステータスが完了以外でも完了日を操作できるように見えていた | ステータスが完了のときだけ完了日を扱い、完了以外では完了日をクリアする仕様に整理 |
|   2 | 登録・更新後の画面反映が分かりにくい場合がある             | キャッシュクリア処理や画面再読み込みで確認する運用とした                         |
|   3 | カテゴリ別追加情報がない場合、何も表示されず分かりにくい   | 追加情報が登録されていない旨を表示するよう修正                                   |
|   4 | FAQ候補管理画面の問い合わせ詳細の表示順が分かりにくい      | 問い合わせ内容、追加情報、対応内容、完了日、FAQ候補状態の順に整理                |
|   5 | 依頼者向け確認画面でEnter検索できない                      | 検索UIをフォーム化し、Enterキーでも検索できるよう修正                            |
|   6 | Streamlitの `use_container_width` 非推奨警告が表示される   | `width="stretch"` に置き換え                                                     |

## 11. 残課題

Ver.2時点では、以下を本格運用に向けた残課題とします。

| 課題                 | 内容                                                     |
| -------------------- | -------------------------------------------------------- |
| ログイン認証         | 依頼者本人の問い合わせだけを表示するために必要           |
| 権限管理             | 依頼者、管理部、管理者で表示・更新範囲を分けるために必要 |
| 通知機能             | 期限前、期限超過、ステータス変更時の通知が未実装         |
| FAQ公開ページ        | Ver.2ではFAQ候補管理までを対象としている                 |
| FAQ検索              | 依頼者が問い合わせ前にFAQ検索する機能は未実装            |
| 添付ファイル         | スクリーンショットや証憑資料の添付は未実装               |
| 操作履歴             | 誰がいつ更新したかの履歴管理は未実装                     |
| 本格承認ワークフロー | 承認フローは仕様が複雑化するため未実装                   |
| 複数ユーザー同時編集 | SQLite・Streamlit構成では本格的な同時編集には制約がある  |

## 12. テスト結果まとめ

Ver.1の主要機能であるCSV取込、問い合わせ一覧表示、新規登録、ステータス更新、期限超過判定、集計、Tableau用CSV出力が動作することを確認しました。
また、Ver.2で追加した要対応アラート、FAQ候補管理、カテゴリ別入力フォーム、依頼者向け確認画面、Ver.2追加集計、Tableau出力列拡張についても、正常に動作することを確認しました。Ver.2追加機能によって既存機能が壊れていないことも確認済みです。

現時点で、ポートフォリオ用の簡易DXアプリとして、問い合わせ管理業務の効率化、対応漏れ防止、確認往復削減、FAQ候補蓄積、依頼者向け状況確認を説明できる状態になっています。
