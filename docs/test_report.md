# テスト結果報告書

## 1. テスト目的

社内問い合わせ管理システムについて、主要機能が想定通り動作することを確認する。
確認対象は以下である。

- CSV取込
- SQLite保存
- 問い合わせ一覧表示
- 新規登録
- ステータス更新
- 期限超過判定
- 対応日数計算
- 集計表示
- Tableau用CSV出力
- Tableau接続

## 2. テスト方法

以下の2種類のテストを実施する。

| 種別           | 内容                                                   |
| -------------- | ------------------------------------------------------ |
| 手動テスト     | Streamlit画面を操作し、登録・更新・集計・CSV出力を確認 |
| スモークテスト | `python -m src.smoke_test` で主要処理の実行確認        |

## 3. 手動テスト結果

詳細は以下のチェックリストに記録する。

```text
docs/test_checklist.md
```

主な確認項目は以下である。

|  No | 確認項目                                           | 結果 |
| --: | -------------------------------------------------- | ---- |
|   1 | CSVをSQLiteに取り込める                            | OK   |
|   2 | Streamlitで問い合わせ一覧を表示できる              | OK   |
|   3 | 部署・カテゴリ・担当者・ステータスで絞り込みできる | OK   |
|   4 | 新規問い合わせを登録できる                         | OK   |
|   5 | 必須入力チェックが動作する                         | OK   |
|   6 | 既存問い合わせを更新できる                         | OK   |
|   7 | ステータスが完了のときだけ完了日を保存できる       | OK   |
|   8 | 完了以外のステータスでは完了日が保存されない       | OK   |
|   9 | 期限超過案件を抽出できる                           | OK   |
|  10 | 対応日数を計算できる                               | OK   |

## 4. 修正した不具合

|  No | 不具合                                                     | 対応                                                                                     |
| --: | ---------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
|   1 | ステータスが完了以外でも完了日を操作できるように見えていた | ステータスが完了のときだけ完了日入力欄を表示し、完了以外では完了日をクリアする仕様に修正 |
|   2 | 登録・更新後の画面反映が分かりにくい場合がある             | キャッシュクリア処理を実装し、必要に応じて画面再読み込みで確認する運用とした             |

## 5. スモークテスト

以下を実行する。

```bash
python -m src.smoke_test
```

確認内容は以下である。

- 問い合わせデータが存在すること
- 必須列が存在すること
- request_id が一意であること
- 派生列が作成されること
- 基本KPIが計算されること
- 集計処理が動作すること
- Tableau出力用DataFrameが作成されること

## 6. 残課題

初期版では以下を対象外とする。

- ログイン認証
- 権限管理
- 通知機能
- FAQ検索
- AI分類
- 本格承認ワークフロー
- 複数ユーザー同時編集への厳密対応

## 7. テスト結果まとめ

主要機能であるCSV取込、問い合わせ一覧表示、新規登録、ステータス更新、期限超過判定、集計、Tableau用CSV出力が動作することを確認した。
初期版として、管理部内で問い合わせ状況を一元管理し、問い合わせ管理作業の効率化を説明できる状態になっている。

## Ver.2 テスト結果

### 実施内容

Ver.2で追加した以下の機能について、構文確認、DB確認、スモークテスト、Streamlit画面確認を実施した。

- 要対応アラート
- FAQ候補管理
- カテゴリ別入力フォーム
- 依頼者向け確認画面
- Ver.2集計
- Tableau出力列拡張

### 実行コマンド

```bash
python -m py_compile app.py src/*.py
python -m src.migrate_db
python -m src.check_db
python -m src.smoke_test
python -m src.smoke_test_ver2
python -m src.export_tableau_csv
streamlit run app.py
```

### 結果

主要機能が正常に動作することを確認した。 また、Ver.2追加機能によって既存の問い合わせ一覧、新規登録、ステータス更新、集計、CSV出力が壊れていないことを確認した。

### WBS8での修正内容

- カテゴリ別追加情報がない場合も、その旨を表示するように修正
- FAQ候補管理画面の問い合わせ詳細表示順を調整
- 依頼者向け確認画面の検索UIをフォーム化し、Enterキーでも検索できるように修正
- Streamlitの `use_container_width` 非推奨警告に対応
