# Ver.3 画面設計書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システム Ver.3 で追加・変更する画面構成を定義するものである。

Ver.3では、Ver.2.0.1までに実装した問い合わせ管理、要対応アラート、FAQ候補管理、カテゴリ別入力フォーム、依頼者向け確認、集計・CSV出力を前提とし、以下の機能を画面上で利用できるようにする。

| 目的                 | 画面上で実現すること                                       |
| -------------------- | ---------------------------------------------------------- |
| 依頼者の自己解決     | FAQ検索画面から公開FAQを検索・閲覧できる                   |
| 依頼者の進捗確認     | ログインユーザー本人の問い合わせだけ確認できる             |
| 対応漏れ防止         | 通知対象確認画面で期限前・期限超過などを確認できる         |
| 更新経緯の可視化     | 履歴確認画面で操作履歴・ステータス履歴を確認できる         |
| 管理者による運用管理 | FAQ公開管理、履歴確認、集計出力を行える                    |
| ロール別利用         | 依頼者、管理部担当者、管理者、閲覧者で表示画面を切り替える |

---

## 2. 画面設計の基本方針

Ver.3の画面設計では、以下を基本方針とする。

| 方針                     | 内容                                                                 |
| ------------------------ | -------------------------------------------------------------------- |
| ロール別に画面を分ける   | 利用者の役割に応じて表示画面・操作範囲を切り替える                   |
| FAQ検索を入口にする      | 依頼者が問い合わせ前に自己解決を試せる導線を作る                     |
| 既存画面を活かす         | Ver.2までの新規登録、更新、アラート、FAQ候補管理、集計画面を拡張する |
| 内部情報を分離する       | 依頼者には管理部内部メモ、操作履歴、内部コメントを表示しない         |
| 操作後の流れを明確にする | 登録、更新、FAQ公開、通知文生成などの処理後に結果を表示する          |
| 実装しすぎない           | 外部メール送信、本格認証、添付ファイル管理は画面設計の対象外とする   |

---

## 3. 共通レイアウト

Ver.3では、全画面に共通して以下の構成を持たせる。

| 領域                 | 内容                                                     |
| -------------------- | -------------------------------------------------------- |
| ヘッダー             | システム名、バージョン、ログイン中ユーザー情報を表示する |
| サイドバーまたはタブ | ロールに応じたメニューを表示する                         |
| メイン領域           | 選択された画面の内容を表示する                           |
| メッセージ領域       | 登録完了、更新完了、エラー、注意事項を表示する           |

## 3.1 ヘッダー表示

ログイン後、画面上部またはサイドバーに以下を表示する。

| 表示項目     | 内容                                               |
| ------------ | -------------------------------------------------- |
| ユーザー名   | ログイン中の利用者名                               |
| 所属部署     | 利用者の所属部署                                   |
| ロール       | `requester`, `staff`, `admin`, `viewer` のいずれか |
| ユーザー切替 | デモ用に別ユーザーへ切り替えるボタンまたは選択欄   |

---

## 4. ロール別メニュー

Ver.3では、ログイン中ユーザーのロールに応じて利用できる画面を切り替える。

| 画面                 | requester | staff | admin | viewer |
| -------------------- | --------: | ----: | ----: | -----: |
| ログイン画面         |         ○ |     ○ |     ○ |      ○ |
| 依頼者トップ         |         ○ |     - |     - |      - |
| FAQ検索              |         ○ |     ○ |     ○ |      - |
| FAQ詳細              |         ○ |     ○ |     ○ |      - |
| 新規問い合わせ       |         ○ |     ○ |     ○ |      - |
| 自分の問い合わせ一覧 |         ○ |     - |     - |      - |
| 問い合わせ一覧       |         - |     ○ |     ○ |      - |
| 問い合わせ更新       |         - |     ○ |     ○ |      - |
| 要対応アラート       |         - |     ○ |     ○ |      - |
| 通知対象確認         |         - |     ○ |     ○ |      - |
| FAQ管理              |         - |     △ |     ○ |      - |
| 履歴確認             |         - |     - |     ○ |      - |
| 集計・CSV出力        |         - |     - |     ○ |      ○ |

凡例は以下とする。

| 記号 | 意味         |
| ---- | ------------ |
| ○    | 利用可能     |
| △    | 一部利用可能 |
| -    | 利用不可     |

`staff` のFAQ管理は、FAQ候補登録・編集案の作成までとし、公開・非公開の最終判断は `admin` が行う。

|  13 | 集計・CSV出力画面    | admin, viewer           | Ver.1〜Ver.3の集計とTableau用CSV出力を行う               |

## 6. ログイン画面

### 6.1 目的

利用者を識別し、ロール別の画面表示に切り替える。
Ver.3では本格的なパスワード認証は実装しない。ポートフォリオ用の簡易ログインとして、ユーザー選択式でログイン状態を作る。

### 6.2 表示項目

| 項目           | 内容                                     |
| -------------- | ---------------------------------------- |
| ユーザー選択   | `users` テーブルの有効ユーザーを選択する |
| ユーザー情報   | 選択ユーザーの部署・ロールを表示する     |
| ログインボタン | 選択ユーザーとしてログインする           |
| 注意書き       | デモ用の簡易ログインであることを表示する |

### 6.3 操作

| 操作         | 処理                                                |
| ------------ | --------------------------------------------------- |
| ユーザー選択 | `users` から選択肢を表示する                        |
| ログイン     | `st.session_state` にログインユーザー情報を保存する |
| ユーザー切替 | セッション情報を更新し、メニューを再描画する        |
| ログアウト   | セッション情報を削除する                            |

### 6.4 遷移

| 条件                | 遷移先                             |
| ------------------- | ---------------------------------- |
| requesterでログイン | 依頼者トップ                       |
| staffでログイン     | 要対応アラートまたは問い合わせ一覧 |
| adminでログイン     | 問い合わせ一覧または管理者メニュー |
| viewerでログイン    | 集計・CSV出力画面                  |

---

## 7. 依頼者トップ

### 7.1 目的

依頼者が最初に利用する入口画面として、FAQ検索、新規問い合わせ、自分の問い合わせ確認への導線を集約する。

### 7.2 表示項目

| 項目                 | 内容                                       |
| -------------------- | ------------------------------------------ |
| 案内文               | 問い合わせ前にFAQ検索を推奨する案内        |
| FAQ検索導線          | FAQ検索画面へのリンクまたはボタン          |
| 新規問い合わせ導線   | 新規問い合わせ画面へのリンクまたはボタン   |
| 自分の問い合わせ導線 | 自分の問い合わせ一覧へのリンクまたはボタン |
| 最近の問い合わせ概要 | 自分の直近問い合わせを数件表示する         |

### 7.3 操作

| 操作                   | 処理                           |
| ---------------------- | ------------------------------ |
| FAQ検索へ進む          | FAQ検索画面を表示する          |
| 新規問い合わせへ進む   | 新規問い合わせ画面を表示する   |
| 自分の問い合わせを見る | 自分の問い合わせ一覧を表示する |

---

## 8. FAQ検索画面

### 8.1 目的

依頼者が問い合わせ前に公開FAQを検索し、自己解決できるようにする。

### 8.2 表示項目

| 項目           | 内容                                               |
| -------------- | -------------------------------------------------- |
| キーワード入力 | FAQタイトル・回答本文を検索する                    |
| カテゴリ選択   | FAQカテゴリで絞り込む                              |
| 検索ボタン     | 条件に一致するFAQを表示する                        |
| FAQ一覧        | FAQタイトル、カテゴリ、更新日、閲覧数を表示する    |
| 未解決時の案内 | 解決しない場合は新規問い合わせへ進む導線を表示する |

### 8.3 操作

| 操作                 | 処理                                                     |
| -------------------- | -------------------------------------------------------- |
| キーワード検索       | `faq_items.title` と `faq_items.answer` を対象に検索する |
| カテゴリ絞り込み     | `faq_items.category` で絞り込む                          |
| FAQ選択              | FAQ詳細画面を表示する                                    |
| 新規問い合わせへ進む | 新規問い合わせ画面を表示する                             |

### 8.4 表示条件

| 条件            | 内容                                         |
| --------------- | -------------------------------------------- |
| 公開FAQのみ表示 | `is_public = 1` のFAQのみ表示する            |
| 非公開FAQ       | requesterには表示しない                      |
| 内部情報        | 元問い合わせの内部メモや個人情報は表示しない |

---

## 9. FAQ詳細画面

### 9.1 目的

FAQの内容を確認し、問い合わせ前の自己解決を支援する。

### 9.2 表示項目

| 項目           | 内容                                 |
| -------------- | ------------------------------------ |
| FAQタイトル    | FAQのタイトル                        |
| カテゴリ       | FAQカテゴリ                          |
| 回答内容       | FAQ回答本文                          |
| 最終更新日     | FAQの最終更新日時                    |
| 役立ちボタン   | FAQが役に立った場合に押す            |
| 問い合わせ導線 | 解決しない場合の新規問い合わせボタン |

### 9.3 操作

| 操作               | 処理                         |
| ------------------ | ---------------------------- |
| FAQ詳細を開く      | `view_count` を加算する      |
| 役立ちボタンを押す | `helpful_count` を加算する   |
| 問い合わせへ進む   | 新規問い合わせ画面へ遷移する |

---

## 10. 新規問い合わせ画面

### 10.1 目的

依頼者または管理部担当者が問い合わせを登録する。Ver.2のカテゴリ別入力フォームを継続利用し、問い合わせ登録時に必要情報を入力できるようにする。

### 10.2 表示項目

| 項目               | 内容                                           |
| ------------------ | ---------------------------------------------- |
| 依頼者名           | ログインユーザー名を初期値として表示する       |
| 部署               | ログインユーザーの部署を初期値として表示する   |
| 受付経路           | フォーム、電話、メールなど                     |
| カテゴリ           | PC・システム、アカウント・権限、経費・請求など |
| サブカテゴリ       | 必要に応じて選択                               |
| 問い合わせ内容     | 自由記述                                       |
| 優先度             | 高・中・低など                                 |
| 希望期限           | 対応希望日                                     |
| カテゴリ別追加項目 | PC管理番号、対象システム、対象日、金額など     |
| 登録ボタン         | 問い合わせを登録する                           |

### 10.3 操作

| 操作         | 処理                                                       |
| ------------ | ---------------------------------------------------------- |
| カテゴリ選択 | カテゴリに応じた追加項目を表示する                         |
| 登録         | `inquiries` に問い合わせを登録する                         |
| 登録後       | 操作履歴を保存し、必要に応じて初期ステータス履歴を保存する |
| 登録完了表示 | 問い合わせIDを画面に表示する                               |

### 10.4 Ver.3での変更点

| 項目                 | Ver.2                  | Ver.3                                |
| -------------------- | ---------------------- | ------------------------------------ |
| 依頼者名             | 手入力                 | ログインユーザー情報を初期値にする   |
| 自分の問い合わせ確認 | IDまたは依頼者名で検索 | ログインユーザーに紐づいて表示       |
| 操作履歴             | なし                   | 作成ログを保存する                   |
| コメント履歴         | なし                   | 必要に応じて登録後にコメント追加可能 |

---

## 11. 自分の問い合わせ一覧

### 11.1 目的

依頼者が、自分が登録した問い合わせの状況を確認できるようにする。

### 11.2 表示項目

| 項目           | 内容                                 |
| -------------- | ------------------------------------ |
| 問い合わせID   | `request_id`                         |
| 受付日         | `request_date`                       |
| カテゴリ       | `category`                           |
| 問い合わせ概要 | `detail` の一部                      |
| ステータス     | `status`                             |
| 回答概要       | `response_summary`                   |
| 完了日         | `completed_date`                     |
| 表示対象       | `requester_visible = 1` のものを表示 |

### 11.3 操作

| 操作     | 処理                                         |
| -------- | -------------------------------------------- |
| 一覧表示 | ログインユーザー本人の問い合わせだけ表示する |
| 詳細表示 | 選択した問い合わせの依頼者向け詳細を表示する |
| 絞り込み | ステータス、カテゴリで絞り込む               |

### 11.4 非表示項目

依頼者画面では、以下を表示しない。

| 非表示項目           | 理由                       |
| -------------------- | -------------------------- |
| 管理部内部メモ       | 内部管理情報のため         |
| 内部コメント         | 依頼者向けではないため     |
| 操作履歴             | 内部監査用情報のため       |
| 担当者の詳細作業メモ | 内部管理情報のため         |
| 非公開FAQ候補        | 管理部側の確認前情報のため |

---

## 12. 問い合わせ一覧画面

### 12.1 目的

管理部担当者または管理者が、問い合わせ全体または担当案件を確認する。

### 12.2 表示項目

| 項目           | 内容                                                                       |
| -------------- | -------------------------------------------------------------------------- |
| KPIカード      | 総件数、未完了件数、期限超過件数、完了件数など                             |
| 絞り込み条件   | 部署、カテゴリ、担当者、ステータス、優先度、期限超過                       |
| 問い合わせ一覧 | 問い合わせID、受付日、依頼者、部署、カテゴリ、担当者、ステータス、期限など |
| 詳細表示       | 追加情報、対応概要、管理項目を確認する                                     |

### 12.3 ロール別表示

| ロール    | 表示範囲                                                 |
| --------- | -------------------------------------------------------- |
| staff     | 自分の担当案件を中心に表示し、必要に応じて全件も確認可能 |
| admin     | 全件表示                                                 |
| requester | 利用不可                                                 |
| viewer    | 利用不可                                                 |

---

## 13. 問い合わせ更新画面

### 13.1 目的

管理部担当者または管理者が、問い合わせの対応状況を更新する。

### 13.2 表示項目

| 項目               | 内容                                           |
| ------------------ | ---------------------------------------------- |
| 対象問い合わせ選択 | 問い合わせID・依頼者・カテゴリなどで選択       |
| 現在情報           | 現在のステータス、担当者、期限、対応概要を表示 |
| ステータス         | 未対応、対応中、情報待ち、完了など             |
| 担当者             | 管理部担当者を選択                             |
| 対応概要           | 依頼者向けに表示可能な回答概要                 |
| 完了日             | 完了時に入力                                   |
| コメント           | 内部向けまたは依頼者向けのコメントを追加       |
| 更新ボタン         | 変更内容を保存する                             |

### 13.3 操作

| 操作           | 処理                                                             |
| -------------- | ---------------------------------------------------------------- |
| ステータス変更 | `inquiries.status` を更新し、`status_history` に履歴を保存する   |
| 担当者変更     | `inquiries.assignee` を更新し、`operation_logs` に履歴を保存する |
| 対応概要更新   | `response_summary` を更新する                                    |
| コメント追加   | `inquiry_comments` にコメントを保存する                          |
| 完了処理       | `completed_date` を保存し、必要に応じて完了通知文を生成する      |

---

## 14. 要対応アラート画面

### 14.1 目的

対応漏れや遅延リスクのある問い合わせを確認する。

### 14.2 表示項目

| アラート種別   | 内容                                       |
| -------------- | ------------------------------------------ |
| 期限超過       | 期限を過ぎた未完了問い合わせ               |
| 本日期限       | 当日が期限の問い合わせ                     |
| 期限間近       | 一定日数以内に期限が来る問い合わせ         |
| 担当者未設定   | 担当者が割り当てられていない問い合わせ     |
| 情報待ち長期化 | 情報待ち状態が一定期間続いている問い合わせ |

### 14.3 操作

| 操作             | 処理                                   |
| ---------------- | -------------------------------------- |
| アラート種別選択 | 種別ごとに対象問い合わせを表示する     |
| 対象確認         | 問い合わせ更新画面へ進む               |
| 通知対象へ送る   | 通知対象確認画面で通知文生成対象にする |

---

## 15. 通知対象確認画面

### 15.1 目的

期限前、期限超過、担当者未設定、情報待ち長期化などの通知対象を確認し、通知文を生成する。Ver.3では、外部メールやSlackへの実送信は行わない。通知対象抽出、通知文生成、通知ログ保存までを対象とする。

### 15.2 表示項目

| 項目           | 内容                                               |
| -------------- | -------------------------------------------------- |
| 通知種別       | 期限前、期限超過、担当者未設定、情報待ち長期化など |
| 対象問い合わせ | 問い合わせID、カテゴリ、担当者、期限、ステータス   |
| 通知対象者     | 担当者、管理者、依頼者など                         |
| 通知文面       | 自動生成された通知文                               |
| 作成日時       | 通知文生成日時                                     |
| 通知状態       | created, reviewed, skipped など                    |

### 15.3 操作

| 操作           | 処理                              |
| -------------- | --------------------------------- |
| 通知対象抽出   | 条件に合う問い合わせを抽出する    |
| 通知文生成     | 通知種別に応じた文面を生成する    |
| 通知ログ保存   | `notification_logs` に保存する    |
| 通知対象除外   | 不要な通知を `skipped` として扱う |
| 確認済みにする | 通知状態を `reviewed` に変更する  |

---

## 16. FAQ管理画面

### 16.1 目的

FAQ候補を公開FAQとして整備し、依頼者向けに公開・非公開を管理する。

### 16.2 表示項目

| 項目        | 内容                        |
| ----------- | --------------------------- |
| FAQ候補一覧 | Ver.2で登録されたFAQ候補    |
| 公開FAQ一覧 | `faq_items` に登録されたFAQ |
| FAQタイトル | FAQとして公開するタイトル   |
| FAQ回答     | 依頼者向けに整えた回答文    |
| カテゴリ    | FAQカテゴリ                 |
| 公開状態    | 公開・非公開                |
| 閲覧数      | FAQ詳細画面を開いた回数     |
| 役立ち件数  | 役立ちボタンが押された回数  |

### 16.3 操作

## 21. 画面設計上の注意点

### 21.1 依頼者に表示しない情報

依頼者には以下を表示しない。

| 非表示情報           | 理由                             |
| -------------------- | -------------------------------- |
| 操作履歴             | 内部管理用のため                 |
| 内部コメント         | 管理部内の対応メモであるため     |
| 担当者の詳細作業メモ | 内部管理情報のため               |
| 非公開FAQ            | 管理者確認前の情報であるため     |
| 通知対象確認画面     | 管理部側の対応管理情報であるため |

### 21.2 FAQ公開時の注意点

FAQは依頼者向けに公開されるため、以下に注意する。

| 注意点       | 内容                                                |
| ------------ | --------------------------------------------------- |
| 個人名       | 必要に応じて一般化する                              |
| 部署固有情報 | そのまま公開してよいか確認する                      |
| 内部判断     | 管理部内部の判断理由は載せない                      |
| 回答文       | 問い合わせ対応文ではなく、再利用可能なFAQ文に整える |

### 21.3 通知機能の注意点

Ver.3では外部通知の実送信は行わない。画面上では、通知対象と通知文を確認できるところまでを実装する。

## 23. 結論

Ver.3では、簡易ログイン、FAQ検索、依頼者トップ、自分の問い合わせ一覧、通知対象確認、履歴確認を追加する。また、Ver.2までに作成した問い合わせ登録、更新、アラート、FAQ候補管理、集計画面を、Ver.3のロール管理・履歴管理・FAQ公開・通知対象抽出に対応する形へ拡張する。

画面設計の中心は、単に画面を増やすことではない。依頼者、管理部担当者、管理者、閲覧者が、それぞれ必要な情報だけを確認し、必要な操作だけを行えるようにすることである。
これにより、問い合わせ管理アプリを、管理部内の効率化ツールから、依頼者も含めた業務システムへ発展させる。
