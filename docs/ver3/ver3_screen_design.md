# Ver.3 画面設計書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システム Ver.3 で追加・変更する画面構成を定義するものである。

Ver.3は、本格的な問い合わせ管理SaaSや社内業務システムを置き換えるものではない。本システムは、メール・Excel・口頭・チャットで分散していた社内問い合わせ対応を題材に、業務フロー整理、データ構造化、FAQ化、対応漏れ防止、履歴管理、集計・可視化を試作するポートフォリオ用PoCである。
そのため、Ver.3の画面設計では、すべての画面を本格運用レベルで作り込むのではなく、以下の区分で整理する。

| 区分     | 方針                                    |
| -------- | --------------------------------------- |
| 必須実装 | Ver.3の中心機能として実装する画面       |
| 簡易実装 | 実務感を示すために最低限実装する画面    |
| 継続     | Ver.2までの画面を維持・一部拡張する画面 |
| 対象外   | Ver.3では作らない画面                   |

Ver.3画面設計の目的は、以下である。

| 目的               | 画面上で実現すること                                       | 実装区分 |
| ------------------ | ---------------------------------------------------------- | -------- |
| 依頼者の自己解決   | FAQ検索画面から公開FAQを検索・閲覧できる                   | 必須     |
| 依頼者の進捗確認   | ログインユーザー本人の問い合わせだけ確認できる             | 必須     |
| ロール別利用       | 依頼者、管理部担当者、管理者、閲覧者で表示画面を切り替える | 必須     |
| 対応経緯の記録     | コメント履歴・ステータス履歴を確認できる                   | 必須     |
| 対応漏れ候補の把握 | 通知対象確認画面で期限前・期限超過などを確認できる         | 簡易     |
| 更新操作の追跡     | 主要操作の履歴を管理者が確認できる                         | 簡易     |
| 改善効果の確認     | FAQ、通知、履歴に関する簡易集計を確認できる                | 簡易     |

---

## 2. 画面設計の基本方針

Ver.3の画面設計では、以下を基本方針とする。

| 方針                   | 内容                                                                   |
| ---------------------- | ---------------------------------------------------------------------- |
| PoCとして設計する      | 本格業務システムではなく、業務改善の試作として画面を設計する           |
| ロール別に画面を分ける | 利用者の役割に応じて表示画面・操作範囲を切り替える                     |
| FAQ検索を入口にする    | 依頼者が問い合わせ前に自己解決を試せる導線を作る                       |
| 既存画面を活かす       | Ver.2までの新規登録、更新、アラート、FAQ候補管理、集計画面を拡張する   |
| 内部情報を分離する     | 依頼者には管理部内部メモ、操作履歴、内部コメントを表示しない           |
| 履歴は簡易に留める     | ステータス履歴・コメント履歴は残すが、完全な監査ログは作らない         |
| 通知は実送信しない     | 通知対象抽出・通知文生成・ログ保存までとする                           |
| 実装しすぎない         | 本格認証、外部通知、添付管理、承認ワークフローは画面設計の対象外とする |

## 3. 共通レイアウト

Ver.3では、全画面に共通して以下の構成を持たせる。

| 領域                 | 内容                                                     |
| -------------------- | -------------------------------------------------------- |
| ヘッダー             | システム名、バージョン、ログイン中ユーザー情報を表示する |
| サイドバーまたはタブ | ロールに応じたメニューを表示する                         |
| メイン領域           | 選択された画面の内容を表示する                           |
| メッセージ領域       | 登録完了、更新完了、エラー、注意事項を表示する           |

### 3.1 ヘッダー表示

ログイン後、画面上部またはサイドバーに以下を表示する。

| 表示項目     | 内容                                               |
| ------------ | -------------------------------------------------- |
| ユーザー名   | ログイン中の利用者名                               |
| 所属部署     | 利用者の所属部署                                   |
| ロール       | `requester`, `staff`, `admin`, `viewer` のいずれか |
| ユーザー切替 | デモ用に別ユーザーへ切り替えるボタンまたは選択欄   |
| 注意書き     | 簡易ログインであり、本格認証ではないことを表示する |

## 4. ロール別メニュー

Ver.3では、ログイン中ユーザーのロールに応じて利用できる画面を切り替える。

| 画面                 | 実装区分   | requester | staff | admin | viewer |
| -------------------- | ---------- | --------: | ----: | ----: | -----: |
| ログイン画面         | 必須       |         ○ |     ○ |     ○ |      ○ |
| 依頼者トップ         | 必須       |         ○ |     - |     - |      - |
| FAQ検索              | 必須       |         ○ |     ○ |     ○ |      - |
| FAQ詳細              | 必須       |         ○ |     ○ |     ○ |      - |
| 新規問い合わせ       | 継続・拡張 |         ○ |     ○ |     ○ |      - |
| 自分の問い合わせ一覧 | 必須       |         ○ |     - |     - |      - |
| 問い合わせ一覧       | 継続・拡張 |         - |     ○ |     ○ |      - |
| 問い合わせ更新       | 継続・拡張 |         - |     ○ |     ○ |      - |
| 要対応アラート       | 継続       |         - |     ○ |     ○ |      - |
| 通知対象確認         | 簡易       |         - |     ○ |     ○ |      - |
| FAQ管理              | 必須       |         - |     △ |     ○ |      - |
| 履歴確認             | 簡易       |         - |     - |     ○ |      - |
| 集計・CSV出力        | 継続・拡張 |         - |     - |     ○ |      ○ |

凡例は以下とする。

| 記号 | 意味         |
| ---- | ------------ |
| ○    | 利用可能     |
| △    | 一部利用可能 |
| -    | 利用不可     |

`staff` のFAQ管理は、FAQ候補登録・FAQ案作成までとする。公開・非公開の最終判断は `admin` が行う。

## 6. ログイン画面

### 6.1 実装区分

必須実装。

### 6.2 目的

利用者を識別し、ロール別の画面表示に切り替える。
Ver.3では本格的なパスワード認証は実装しない。ポートフォリオ用の簡易ログインとして、ユーザー選択式でログイン状態を作る。

### 6.3 表示項目

| 項目           | 内容                                                       |
| -------------- | ---------------------------------------------------------- |
| ユーザー選択   | `users` テーブルの有効ユーザーを選択する                   |
| ユーザー情報   | 選択ユーザーの部署・ロールを表示する                       |
| ログインボタン | 選択ユーザーとしてログインする                             |
| 注意書き       | デモ用の簡易ログインであり、本格認証ではないことを表示する |

### 6.4 操作

| 操作         | 処理                                                |
| ------------ | --------------------------------------------------- |
| ユーザー選択 | `users` から選択肢を表示する                        |
| ログイン     | `st.session_state` にログインユーザー情報を保存する |
| ユーザー切替 | セッション情報を更新し、メニューを再描画する        |
| ログアウト   | セッション情報を削除する                            |

### 6.5 遷移

| 条件                | 遷移先                             |
| ------------------- | ---------------------------------- |
| requesterでログイン | 依頼者トップ                       |
| staffでログイン     | 要対応アラートまたは問い合わせ一覧 |
| adminでログイン     | 問い合わせ一覧または管理者メニュー |
| viewerでログイン    | 集計・CSV出力画面                  |

### 6.6 対象外

| 対象外              | 理由                            |
| ------------------- | ------------------------------- |
| パスワード入力      | 本格認証をVer.3対象外とするため |
| パスワードリセット  | 同上                            |
| OAuth・社内認証連携 | PoC範囲を超えるため             |
| ログイン履歴管理    | 本格監査ログを対象外とするため  |

## 7. 依頼者トップ

### 7.1 実装区分

必須実装。

### 7.2 目的

依頼者が最初に利用する入口画面として、FAQ検索、新規問い合わせ、自分の問い合わせ確認への導線を集約する。

### 7.3 表示項目

| 項目                 | 内容                                       |
| -------------------- | ------------------------------------------ |
| 案内文               | 問い合わせ前にFAQ検索を推奨する案内        |
| FAQ検索導線          | FAQ検索画面へのリンクまたはボタン          |
| 新規問い合わせ導線   | 新規問い合わせ画面へのリンクまたはボタン   |
| 自分の問い合わせ導線 | 自分の問い合わせ一覧へのリンクまたはボタン |
| 最近の問い合わせ概要 | 自分の直近問い合わせを数件表示する         |

### 7.4 操作

| 操作                   | 処理                           |
| ---------------------- | ------------------------------ |
| FAQ検索へ進む          | FAQ検索画面を表示する          |
| 新規問い合わせへ進む   | 新規問い合わせ画面を表示する   |
| 自分の問い合わせを見る | 自分の問い合わせ一覧を表示する |

---

## 8. FAQ検索画面

### 8.1 実装区分

必須実装。

### 8.2 目的

依頼者が問い合わせ前に公開FAQを検索し、自己解決できるようにする。

### 8.3 表示項目

| 項目           | 内容                                               |
| -------------- | -------------------------------------------------- |
| キーワード入力 | FAQタイトル・回答本文を検索する                    |
| カテゴリ選択   | FAQカテゴリで絞り込む                              |
| 検索ボタン     | 条件に一致するFAQを表示する                        |
| FAQ一覧        | FAQタイトル、カテゴリ、更新日、閲覧数を表示する    |
| 未解決時の案内 | 解決しない場合は新規問い合わせへ進む導線を表示する |

### 8.4 操作

| 操作                 | 処理                                                     |
| -------------------- | -------------------------------------------------------- |
| キーワード検索       | `faq_items.title` と `faq_items.answer` を対象に検索する |
| カテゴリ絞り込み     | `faq_items.category` で絞り込む                          |
| FAQ選択              | FAQ詳細画面を表示する                                    |
| 新規問い合わせへ進む | 新規問い合わせ画面を表示する                             |

### 8.5 表示条件

| 条件            | 内容                                         |
| --------------- | -------------------------------------------- |
| 公開FAQのみ表示 | `is_public = 1` のFAQのみ表示する            |
| 非公開FAQ       | requesterには表示しない                      |
| 内部情報        | 元問い合わせの内部メモや個人情報は表示しない |

---

## 9. FAQ詳細画面

### 9.1 実装区分

必須実装。

### 9.2 目的

FAQの内容を確認し、問い合わせ前の自己解決を支援する。

### 9.3 表示項目

| 項目           | 内容                                 |
| -------------- | ------------------------------------ |
| FAQタイトル    | FAQのタイトル                        |
| カテゴリ       | FAQカテゴリ                          |
| 回答内容       | FAQ回答本文                          |
| 最終更新日     | FAQの最終更新日時                    |
| 閲覧数         | FAQが閲覧された回数                  |
| 役立ちボタン   | FAQが役に立った場合に押す            |
| 問い合わせ導線 | 解決しない場合の新規問い合わせボタン |

### 9.4 操作

| 操作               | 処理                         |
| ------------------ | ---------------------------- |
| FAQ詳細を開く      | `view_count` を加算する      |
| 役立ちボタンを押す | `helpful_count` を加算する   |
| 問い合わせへ進む   | 新規問い合わせ画面へ遷移する |

### 9.5 作り込まないこと

| 作らないもの           | 理由                                  |
| ---------------------- | ------------------------------------- |
| 閲覧者ごとの詳細ログ   | `view_count` で十分なため             |
| 役立ち評価の複数段階化 | PoCでは「役に立った」件数で十分なため |
| FAQ推薦機能            | AI・検索高度化の領域になるため        |

## 10. 新規問い合わせ画面

### 10.1 実装区分

継続・拡張。

### 10.2 目的

依頼者または管理部担当者が問い合わせを登録する。
Ver.2のカテゴリ別入力フォームを継続利用し、問い合わせ登録時に必要情報を入力できるようにする。

### 10.3 表示項目

| 項目               | 内容                                           |
| ------------------ | ---------------------------------------------- |
| 依頼者名           | ログインユーザー名を初期値として表示する       |
| 部署               | ログインユーザーの部署を初期値として表示する   |
| 受付経路           | フォーム、電話、メールなど                     |
| カテゴリ           | PC・システム、アカウント・権限、経費・請求など |
| サブカテゴリ       | 必要に応じて選択                               |
| 問い合わせ内容     | 自由記述                                       |
| 優先度             | 高・中・低など                                 |
| 希望期限           | 対応希望日                                     |
| カテゴリ別追加項目 | PC管理番号、対象システム、対象日、金額など     |
| 登録ボタン         | 問い合わせを登録する                           |

### 10.4 操作

| 操作         | 処理                                                           |
| ------------ | -------------------------------------------------------------- |
| カテゴリ選択 | カテゴリに応じた追加項目を表示する                             |
| 登録         | `inquiries` に問い合わせを登録する                             |
| 登録後       | 操作履歴を簡易保存し、必要に応じて初期ステータス履歴を保存する |
| 登録完了表示 | 問い合わせIDを画面に表示する                                   |

### 10.5 Ver.3での変更点

| 項目                 | Ver.2                  | Ver.3                              |
| -------------------- | ---------------------- | ---------------------------------- |
| 依頼者名             | 手入力                 | ログインユーザー情報を初期値にする |
| 自分の問い合わせ確認 | IDまたは依頼者名で検索 | ログインユーザーに紐づいて表示     |
| 操作履歴             | なし                   | 主要操作のみ簡易保存               |
| ステータス履歴       | なし                   | 初期ステータスを必要に応じて保存   |

## 11. 自分の問い合わせ一覧

### 11.1 実装区分

必須実装。

### 11.2 目的

依頼者が、自分が登録した問い合わせの状況を確認できるようにする。

### 11.3 表示項目

| 項目               | 内容                                        |
| ------------------ | ------------------------------------------- |
| 問い合わせID       | `request_id`                                |
| 受付日             | `request_date`                              |
| カテゴリ           | `category`                                  |
| 問い合わせ概要     | `detail` の一部                             |
| ステータス         | `status`                                    |
| 回答概要           | `response_summary`                          |
| 完了日             | `completed_date`                            |
| 依頼者向けコメント | `visibility = requester` のコメントのみ表示 |
| 表示対象           | `requester_visible = 1` のものを表示        |

### 11.4 操作

| 操作     | 処理                                         |
| -------- | -------------------------------------------- |
| 一覧表示 | ログインユーザー本人の問い合わせだけ表示する |
| 詳細表示 | 選択した問い合わせの依頼者向け詳細を表示する |
| 絞り込み | ステータス、カテゴリで絞り込む               |

### 11.5 非表示項目

依頼者画面では、以下を表示しない。

| 非表示項目           | 理由                           |
| -------------------- | ------------------------------ |
| 管理部内部メモ       | 内部管理情報のため             |
| 内部コメント         | 依頼者向けではないため         |
| 操作履歴             | 内部管理用情報のため           |
| ステータス履歴の詳細 | 管理部内部の対応経緯であるため |
| 担当者の詳細作業メモ | 内部管理情報のため             |
| 非公開FAQ候補        | 管理部側の確認前情報のため     |

## 12. 問い合わせ一覧画面

### 12.1 実装区分

継続・拡張。

### 12.2 目的

管理部担当者または管理者が、問い合わせ全体または担当案件を確認する。

### 12.3 表示項目

| 項目           | 内容                                                                       |
| -------------- | -------------------------------------------------------------------------- |
| KPIカード      | 総件数、未完了件数、期限超過件数、完了件数など                             |

### 12.4 ロール別表示

| ロール    | 表示範囲                                                 |
| --------- | -------------------------------------------------------- |
| staff     | 自分の担当案件を中心に表示し、必要に応じて全件も確認可能 |
| admin     | 全件表示                                                 |
| requester | 利用不可                                                 |
| viewer    | 利用不可                                                 |

## 13. 問い合わせ更新画面

### 13.1 実装区分

継続・拡張。

### 13.2 目的

管理部担当者または管理者が、問い合わせの対応状況を更新する。

### 13.3 表示項目

| 項目               | 内容                                           |
| ------------------ | ---------------------------------------------- |
| 対象問い合わせ選択 | 問い合わせID・依頼者・カテゴリなどで選択       |
| 現在情報           | 現在のステータス、担当者、期限、対応概要を表示 |
| ステータス         | 未対応、対応中、情報待ち、完了など             |
| 担当者             | 管理部担当者を選択                             |
| 対応概要           | 依頼者向けに表示可能な回答概要                 |
| 完了日             | 完了時に入力                                   |
| コメント           | 内部向けまたは依頼者向けのコメントを追加       |
| 更新ボタン         | 変更内容を保存する                             |

### 13.4 操作

| 操作           | 処理                                                                 |
| -------------- | -------------------------------------------------------------------- |

## 22. 画面設計上の注意点

### 22.1 依頼者に表示しない情報

依頼者には以下を表示しない。

| 非表示情報           | 理由                             |
| -------------------- | -------------------------------- |
| 操作履歴             | 内部管理用のため                 |
| 内部コメント         | 管理部内の対応メモであるため     |
| 担当者の詳細作業メモ | 内部管理情報のため               |
| 非公開FAQ            | 管理者確認前の情報であるため     |
| 通知対象確認画面     | 管理部側の対応管理情報であるため |
| ステータス履歴の詳細 | 内部対応経緯であるため           |

### 22.2 FAQ公開時の注意点

FAQは依頼者向けに公開されるため、以下に注意する。

| 注意点       | 内容                                                |
| ------------ | --------------------------------------------------- |
| 個人名       | 必要に応じて一般化する                              |
| 部署固有情報 | そのまま公開してよいか確認する                      |
| 内部判断     | 管理部内部の判断理由は載せない                      |
| 回答文       | 問い合わせ対応文ではなく、再利用可能なFAQ文に整える |

### 22.3 通知機能の注意点

Ver.3では外部通知の実送信は行わない。画面上では、通知対象と通知文を確認できるところまでを実装する。

### 22.4 履歴機能の注意点

Ver.3の履歴機能は、対応経緯を確認するための簡易履歴である。内部統制・監査証跡・アクセスログ管理を目的とした本格機能ではない。

## 24. 結論

Ver.3では、簡易ログイン、依頼者トップ、FAQ検索、FAQ詳細、自分の問い合わせ一覧、FAQ管理を中心に実装する。また、Ver.2までに作成した問い合わせ登録、更新、アラート、集計画面を、Ver.3のロール管理・履歴管理・FAQ公開に対応する形へ拡張する。
通知対象確認、履歴確認、Ver.3集計は、業務改善効果を説明するための簡易画面として実装する。外部メール送信、Slack / Teams連携、本格認証、添付ファイル管理、承認ワークフロー、詳細監査ログ、AI分類、本格BIダッシュボードはVer.3では作らない。
画面設計の中心は、単に画面を増やすことではない。依頼者、管理部担当者、管理者、閲覧者が、それぞれ必要な情報だけを確認し、必要な操作だけを行えるようにすることである。
これにより、問い合わせ管理アプリを、管理部内の効率化ツールから、依頼者の自己解決、対応経緯の可視化、対応漏れ候補の把握、改善分析を試作できるポートフォリオ用PoCへ発展させる。
