# Ver.3 WBS

## 1. 目的

本ドキュメントは、社内問い合わせ管理システム Ver.3 の開発作業をWBSとして整理するものである。

Ver.3では、Ver.2.0.1までに実装した問い合わせ管理、要対応アラート、FAQ候補管理、カテゴリ別入力フォーム、依頼者向け確認、Ver.2追加集計、Tableau出力列拡張を前提とする。
Ver.3の目的は、Ver.2で蓄積できるようになった問い合わせ情報、FAQ候補、アラート情報を、依頼者・管理部担当者・管理者が実際に活用できる運用機能へ接続することである。
また、Ver.3本体機能に入る前に、WBS0として `app.py` のUI分割とアプリ構造整理を行った。本WBSでは、WBS0後の構造を前提に、WBS1以降の作業を定義する。

---

## 2. Ver.3開発方針

Ver.3では、以下を重視する。

| 方針                       | 内容                                                            |
| -------------------------- | --------------------------------------------------------------- |
| 構造整理を前提にする       | WBS0で分割した `src/ui/` 構成を活用する                         |
| Ver.2.0.1機能を壊さない    | 既存の登録、更新、アラート、FAQ候補、依頼者確認、集計を維持する |
| 業務価値を明確にする       | FAQ公開、通知対象抽出、履歴管理により運用改善を示す             |
| 実装範囲を広げすぎない     | 外部送信、AI分類、本格認証、添付管理は対象外とする              |
| 設計と実装を対応させる     | 要件、画面、データ、業務フロー、アーキテクチャに沿って実装する  |
| テストとドキュメントを残す | ポートフォリオとして説明可能な状態にする                        |

## 9. 結論

Ver.3開発では、WBS0で実施した `app.py` のUI分割を前提に、設計、DB拡張、簡易ログイン、FAQ公開、依頼者向け画面、通知対象抽出、履歴管理、集計・CSV出力拡張、テスト、リリース準備を順に進める。

重要なのは、いきなり機能を追加するのではなく、WBS1で設計を固定し、WBS2以降で段階的に実装することである。この進め方により、Ver.2.0.1で完成した機能を維持しながら、Ver.3として「依頼者の自己解決」「対応漏れ防止」「履歴管理」「改善分析」を備えた業務システムへ発展させることができる。
