# 操作手順書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システムの基本操作を説明するものです。

本システムは、Streamlitで作成した問い合わせ管理アプリです。問い合わせの登録、一覧確認、ステータス更新、要対応アラート確認、FAQ候補管理、依頼者向け確認、集計・CSV出力を行います。

Ver.1では、主に以下の画面を対象としていました。

- 問い合わせ一覧
- 新規登録
- ステータス更新
- 集計・CSV出力

Ver.2では、以下の画面・機能を追加しています。

- 要対応アラート
- FAQ候補管理
- カテゴリ別入力フォーム
- 依頼者向け確認
- Ver.2追加集計
- Tableau出力列の拡張

## 2. アプリの起動方法

プロジェクトフォルダで以下を実行します。

```bash
cd ~/projects/inquiry-dx-system
source .venv/bin/activate
streamlit run app.py
```

ブラウザでStreamlitアプリが開きます。
DBが未作成の場合や、CSVからデータを取り込み直す場合は、先に以下を実行します。

```bash
python -m src.import_csv
python -m src.migrate_db
python -m src.check_db
```

## 3. 画面構成

Ver.2時点の主なタブは以下です。

| タブ           | 用途                                                   |
| -------------- | ------------------------------------------------------ |
| 要対応アラート | 優先的に確認すべき問い合わせを確認する                 |
| 問い合わせ一覧 | 登録済み問い合わせを一覧・絞り込みで確認する           |
| 新規登録       | 新しい問い合わせを登録する                             |
| ステータス更新 | 既存問い合わせの担当者・ステータス・対応内容を更新する |
| FAQ候補管理    | 完了済み問い合わせからFAQ候補を登録・更新・解除する    |
| 依頼者向け確認 | 問い合わせIDまたは依頼者名で状況を確認する             |
| 集計・CSV出力  | 問い合わせ状況の集計とTableau用CSV出力を行う           |

## 4. 要対応アラート画面

### 4.1 目的

要対応アラート画面は、優先的に確認すべき問い合わせを把握するための画面です。期限超過や担当者未設定など、対応漏れにつながりやすい問い合わせを一覧で確認できます。

### 4.2 表示されるアラート

| アラート種別   | 内容                             |
| -------------- | -------------------------------- |
| 期限超過       | 未完了で、希望期限を過ぎている   |
| 本日期限       | 未完了で、希望期限が今日         |
| 期限間近       | 未完了で、希望期限が明日まで     |
| 担当者未設定   | 担当者が未設定                   |
| 情報待ち長期化 | 情報待ち状態が一定日数続いている |

### 4.3 操作手順

1. 「要対応アラート」タブを開く
2. アラート対象件数を確認する
3. 期限超過、本日期限、期限間近、担当者未設定、情報待ち長期化の件数を確認する
4. 必要に応じて、アラート種別で一覧を絞り込む
5. 対応が必要な問い合わせIDを確認する
6. 「ステータス更新」タブで担当者・ステータス・対応内容を更新する

## 5. 問い合わせ一覧画面

### 5.1 目的

問い合わせ一覧画面は、登録済みの問い合わせを一覧で確認するための画面です。部署、カテゴリ、担当者、ステータス、優先度、期限超過などの条件で絞り込みできます。

### 5.2 表示される主な項目

| 項目               | 内容                                         |
| ------------------ | -------------------------------------------- |
| 問い合わせID       | 問い合わせを識別するID                       |
| 受付日             | 問い合わせを受け付けた日                     |
| 依頼者             | 問い合わせを行った社員                       |
| 部署               | 依頼者の所属部署                             |
| 受付経路           | フォーム、メール、電話など                   |
| カテゴリ           | 問い合わせ分類                               |
| 小分類             | より細かい分類                               |
| 優先度             | 高・中・低                                   |
| 希望期限           | 対応希望日                                   |
| 担当者             | 管理部担当者                                 |
| ステータス         | 未対応、対応中、情報待ち、承認待ち、完了など |
| 期限超過           | 期限を過ぎているか                           |
| 問い合わせ内容     | 依頼内容の詳細                               |
| カテゴリ別追加情報 | カテゴリ別入力フォームで入力された補足情報   |
| 対応内容           | 管理部の回答・対応内容                       |

### 5.3 絞り込み手順

1. 「問い合わせ一覧」タブを開く
2. 必要に応じて、部署、カテゴリ、担当者、ステータス、優先度を選択する
3. 期限超過のみを確認したい場合は、期限超過条件を指定する
4. 絞り込み後の一覧を確認する
5. 必要に応じて、問い合わせIDを控え、ステータス更新画面で更新する

絞り込みを行うと、一覧だけでなく、画面上のKPIも絞り込み後の値に変わります。

## 6. 新規登録画面

### 6.1 目的

新規登録画面は、新しい問い合わせを登録するための画面です。Ver.2では、カテゴリに応じた追加項目も入力できるようにしています。

### 6.2 基本入力項目

| 項目               | 必須 | 内容                       |
| ------------------ | ---- | -------------------------- |
| 依頼者名           | 必須 | 問い合わせを行った社員名   |
| 部署               | 必須 | 依頼者の所属部署           |
| 受付経路           | 必須 | フォーム、メール、電話など |
| カテゴリ           | 必須 | 問い合わせ分類             |
| 小分類             | 任意 | より細かい分類             |
| 優先度             | 必須 | 高・中・低                 |
| 希望期限           | 必須 | 対応希望日                 |
| 担当者             | 任意 | 管理部担当者               |
| ステータス         | 必須 | 初期状態は未対応           |
| 問い合わせ内容     | 必須 | 問い合わせの詳細           |
| 不足情報・確認事項 | 任意 | 追加で確認が必要な情報     |

### 6.3 カテゴリ別入力フォーム

Ver.2では、カテゴリを選択すると、カテゴリに応じた追加項目が表示されます。

| カテゴリ         | 追加項目                                                 |
| ---------------- | -------------------------------------------------------- |
| PC・システム     | PC管理番号、発生時刻、エラー内容、再起動有無             |
| アカウント・権限 | 対象システム、対象フォルダ・対象機能、必要な権限、承認者 |
| 勤怠・労務       | 対象日、修正前時刻、修正後時刻、理由                     |
| 経費・請求       | 金額、取引先、対象月、添付書類有無                       |
| 備品・設備       | 備品名・設備名、数量、利用目的、希望日                   |

### 6.4 登録手順

1. 「新規登録」タブを開く
2. カテゴリを選択する
3. 依頼者名、部署、受付経路、優先度、希望期限などを入力する
4. 問い合わせ内容を入力する
5. 必要に応じて、不足情報・確認事項を入力する
6. カテゴリ別追加項目を入力する
7. 「登録する」を押す
8. 登録後、問い合わせ一覧画面で内容を確認する

登録すると、問い合わせIDが自動発行されます。

```text
REQ-YYYYMMDD-001
```

## 7. ステータス更新画面

### 7.1 目的

ステータス更新画面は、既存問い合わせの担当者、ステータス、対応内容、作業時間を更新するための画面です。

### 7.2 更新できる項目

| 項目               | 内容                                         |
| ------------------ | -------------------------------------------- |
| 担当者             | 管理部担当者                                 |
| ステータス         | 未対応、対応中、情報待ち、承認待ち、完了など |
| 完了日             | ステータスが完了の場合に設定                 |
| 対応内容           | 管理部の回答・対応内容                       |
| 記録・管理上の問題 | 管理上の補足事項                             |
| 管理作業時間       | 転記、確認、進捗管理などにかかった時間       |
| 実対応時間         | PC確認、権限付与など実対応にかかった時間     |

### 7.3 更新手順

1. 「ステータス更新」タブを開く
2. 更新したい問い合わせを選択する
3. 現在の問い合わせ内容を確認する
4. カテゴリ別追加情報を確認する
   - 追加情報がない場合は、その旨が表示される
5. 担当者、ステータス、対応内容などを更新する
6. 必要に応じて、管理作業時間・実対応時間を入力する
7. 「更新する」を押す
8. 問い合わせ一覧画面で更新内容を確認する

### 7.4 完了日の扱い

完了日は、ステータスが「完了」のときだけ保存されます。ステータスが以下の場合、完了日は保存されません。

- 未対応
- 対応中
- 情報待ち
- 承認待ち

完了済みの問い合わせを完了以外のステータスに戻した場合、完了日はクリアされます。

## 8. FAQ候補管理画面

### 8.1 目的

FAQ候補管理画面は、完了済み問い合わせの中から、よくある問い合わせとしてFAQ化できそうなものを候補登録する画面です。
Ver.2では、FAQ公開ページまでは作らず、FAQ候補を蓄積する段階までを対象としています。

### 8.2 FAQ候補として保存する内容

| 項目        | 内容                          |
| ----------- | ----------------------------- |
| FAQタイトル | FAQとして公開する場合の見出し |
| FAQ回答案   | 依頼者向けに整理した回答案    |

### 8.3 FAQ候補を登録する手順

1. 「FAQ候補管理」タブを開く
2. 完了済み問い合わせを選択する
3. 元の問い合わせ内容を確認する
4. カテゴリ別追加情報を確認する
   - 追加情報がない場合は、その旨が表示される
5. 対応内容、完了日を確認する
6. FAQ候補状態を確認する
   - 現在FAQ候補の場合は、その旨が表示される
   - まだFAQ候補でない場合は、その旨が表示される
7. FAQタイトルを入力する
8. FAQ回答案を入力する
9. 「FAQ候補として保存」または「FAQ候補として保存・更新」を押す
10. FAQ候補一覧に表示されることを確認する

### 8.4 FAQ候補を更新する手順

1. 「FAQ候補管理」タブを開く
2. すでにFAQ候補になっている問い合わせを選択する
3. FAQタイトルまたはFAQ回答案を修正する
4. 「FAQ候補として保存・更新」を押す
5. FAQ候補一覧で更新内容を確認する

既存のFAQ候補情報は上書き更新されます。

### 8.5 FAQ候補から外す手順

1. 「FAQ候補管理」タブを開く
2. FAQ候補になっている問い合わせを選択する
3. 「FAQ候補から外す」を押す
4. FAQ候補一覧から外れることを確認する

FAQ候補から外しても、FAQタイトル・FAQ回答案は下書きとして保持されます。

### 8.6 FAQ候補CSVを出力する手順

1. 「FAQ候補管理」タブを開く
2. FAQ候補一覧を確認する
3. 「FAQ候補CSVをダウンロード」を押す
4. `faq_candidates.csv` がダウンロードされることを確認する

## 9. 依頼者向け確認画面

### 9.1 目的

依頼者向け確認画面は、依頼者が自分の問い合わせ状況を確認するための画面です。問い合わせIDまたは依頼者名で検索し、ステータス、担当者、希望期限、完了日、管理部からの回答などを確認できます。
Ver.2ではデモ画面として実装しており、本格運用ではログイン認証・権限管理が必要です。

### 9.2 問い合わせIDで検索する手順

1. 「依頼者向け確認」タブを開く
2. 検索方法で「問い合わせIDで検索」を選択する
3. 問い合わせIDを入力する
4. Enterキー、または「問い合わせ状況を確認」ボタンで検索する
5. 問い合わせ状況を確認する
6. 必要に応じて、詳細表示を開く

### 9.3 依頼者名で検索する手順

1. 「依頼者向け確認」タブを開く
2. 検索方法で「依頼者名で検索」を選択する
3. 依頼者名を入力する
4. Enterキー、または「問い合わせ状況を確認」ボタンで検索する
5. 該当する問い合わせ一覧を確認する
6. 必要に応じて、詳細表示を開く

### 9.4 表示される主な項目

| 項目                       | 内容                                   |
| -------------------------- | -------------------------------------- |
| 問い合わせID               | 問い合わせを識別するID                 |
| 受付日                     | 問い合わせを受け付けた日               |
| 依頼者                     | 問い合わせを行った社員                 |
| 部署                       | 依頼者の所属部署                       |
| カテゴリ                   | 問い合わせ分類                         |
| 問い合わせ内容             | 依頼内容                               |
| 追加情報                   | カテゴリ別入力フォームで入力された情報 |
| ステータス                 | 現在の対応状況                         |
| 担当者                     | 管理部担当者                           |
| 希望期限                   | 対応希望日                             |
| 完了日                     | 対応完了日                             |
| 管理部からの回答・対応内容 | 回答または対応結果                     |

### 9.5 表示しない情報

依頼者向け確認画面では、以下のような管理部内部向け情報は表示しません。

- 管理作業時間
- 実対応時間
- 記録・管理上の問題
- FAQ候補フラグ
- FAQ回答案の下書き

## 10. 集計・CSV出力画面

### 10.1 目的

集計・CSV出力画面は、問い合わせ状況を集計し、Tableau連携用CSVを出力するための画面です。
Ver.1由来の基本集計に加えて、Ver.2追加機能の集計も確認できます。

### 10.2 表示される基本KPI

| KPI                | 内容                             |
| ------------------ | -------------------------------- |
| 問い合わせ件数     | 対象データの件数                 |
| 未完了件数         | ステータスが完了以外の件数       |
| 期限超過件数       | 未完了で希望期限を過ぎている件数 |
| 平均対応日数       | 受付日から完了日までの平均日数   |
| 管理作業時間       | 管理作業にかかった合計時間       |
| 実対応時間         | 実対応にかかった合計時間         |
| 問い合わせ関連時間 | 管理作業時間と実対応時間の合計   |

### 10.3 表示される基本集計

- 期限超過案件一覧
- カテゴリ別件数
- 担当者別件数
- 部署別件数
- ステータス別件数
- 受付経路別件数
- 優先度別件数
- 担当者別作業時間
- カテゴリ別作業時間
- カテゴリ別平均対応日数

### 10.4 Ver.2追加集計

Ver.2では、以下の集計を追加しています。

| 集計                     | 内容                                     |
| ------------------------ | ---------------------------------------- |
| 要対応アラート件数       | 何らかのアラートに該当する問い合わせ件数 |
| アラート種別別件数       | 期限超過、本日期限、期限間近などの件数   |
| FAQ候補件数              | FAQ候補として登録された問い合わせ件数    |
| カテゴリ別FAQ候補件数    | FAQ候補をカテゴリ別に集計                |
| 追加情報入力件数         | カテゴリ別追加情報が入力されている件数   |
| 追加情報入力率           | 追加情報が入力されている割合             |
| カテゴリ別追加情報入力率 | カテゴリごとの追加情報入力率             |
| 依頼者向け表示件数       | 依頼者向け画面に表示される問い合わせ件数 |
| 依頼者向け非表示件数     | 依頼者向け画面に表示しない問い合わせ件数 |

### 10.5 Tableau用CSVを出力する手順

1. 「集計・CSV出力」タブを開く
2. 集計内容を確認する
3. Tableau用CSV出力ボタンを押す
4. CSVが出力されることを確認する

ターミナルから出力する場合は以下です。

```bash
python -m src.export_tableau_csv
```

出力先は以下です。

```text
data/tableau_output.csv
```

## 7. 終了方法

ターミナルで以下を押します。

```text
Ctrl + C
```

ブラウザのタブを閉じるだけでは、Streamlitプロセスは終了しません。アプリを完全に終了する場合は、ターミナル側で停止してください。

## 12. 操作時の注意点

### 12.1 本格運用では認証が必要

Ver.2の依頼者向け確認画面はデモ用です。本格運用では、ログイン認証を行い、本人の問い合わせだけを表示する必要があります。

### 12.2 FAQ候補は公開FAQではない

Ver.2のFAQ候補管理は、FAQ公開ページではありません。完了済み問い合わせからFAQ化できそうなものを蓄積するための管理機能です。

### 12.3 カテゴリ別追加情報は補助情報である

カテゴリ別入力フォームの追加項目は、確認往復を減らすための補助情報です。未入力でも問い合わせ登録は可能ですが、入力されている方が管理部の確認作業を減らしやすくなります。

### 12.4 更新後は一覧や集計で確認する

新規登録、ステータス更新、FAQ候補登録を行った後は、以下で反映結果を確認してください。

- 問い合わせ一覧
- ステータス更新画面
- FAQ候補一覧
- 集計・CSV出力画面
