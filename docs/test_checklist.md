# テストチェックリスト

## 1. テスト目的

社内問い合わせ管理システムについて、CSV取込、DB保存、一覧表示、新規登録、ステータス更新、集計、Tableau用CSV出力が正常に動作することを確認する。

## 2. テスト対象

- CSV取込
- SQLite保存
- Streamlit一覧表示
- 新規登録
- ステータス更新
- 期限超過判定
- 対応日数計算
- 集計表示
- Tableau用CSV出力
- Tableau接続確認

## 4. 確認した不具合

|  No | 不具合内容                               | 影響 | 対応                                                                                                                                                                                       |
| --: | ---------------------------------------- | ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
|   1 | 登録後に一覧が即時更新されない           | 軽微 | ブラウザ再読み込みで反映。後続でst.rerun検討                                                                                                                                               |
|   2 | 完了日の日付が固定されているのは仕様か。 |      | ステータスが「完了」のときだけ、完了日を入力できる。ステータスが「完了」以外のときは、完了日は入力・保存しない。完了済み案件を「対応中」「情報待ち」などに戻した場合、完了日はクリアする。 |

## 5. テスト結果まとめ

主要機能である問い合わせ一覧表示、新規登録、ステータス更新、集計、Tableau用CSV出力が動作することを確認した。

残課題として、ログイン認証、権限管理、通知機能、FAQ検索、AI分類は初期版では対象外とする。
