# 画面設計書

## 1. 目的

本ドキュメントは、社内問い合わせ管理システムの画面構成と、各画面の役割・表示項目・操作内容を整理するものです。

本システムは、Streamlitで作成した簡易DXアプリです。
Ver.1では、問い合わせ一覧、新規登録、ステータス更新、集計・CSV出力の4画面を中心に設計しました。
Ver.2では、問い合わせ管理機能を拡張し、以下の画面・機能を追加しています。

- 要対応アラート
- FAQ候補管理
- カテゴリ別入力フォーム
- 依頼者向け確認
- Ver.2追加集計
- Tableau出力列の拡張

## 2. 画面構成

Ver.2時点のStreamlitアプリは、以下のタブ構成とします。

| タブ           | 目的                                                   |
| -------------- | ------------------------------------------------------ |
| 要対応アラート | 優先的に確認すべき問い合わせを確認する                 |
| 問い合わせ一覧 | 登録済み問い合わせの一覧確認・絞り込みを行う           |
| 新規登録       | 新しい問い合わせを登録する                             |
| ステータス更新 | 既存問い合わせの担当者・ステータス・対応内容を更新する |
| FAQ候補管理    | 完了済み問い合わせからFAQ候補を登録・更新・解除する    |
| 依頼者向け確認 | 問い合わせIDまたは依頼者名で問い合わせ状況を確認する   |
| 集計・CSV出力  | 問い合わせ状況の集計とTableau用CSV出力を行う           |

## 3. 画面遷移・利用の流れ

基本的な利用の流れは以下です。

```text
新規登録
  ↓
問い合わせ一覧で確認
  ↓
要対応アラートで優先確認
  ↓
ステータス更新
  ↓
完了済み問い合わせをFAQ候補管理で整理
  ↓
依頼者向け確認画面で状況確認
  ↓
集計・CSV出力で状況を可視化
```

管理部担当者は、主に以下の画面を利用します。

- 要対応アラート
- 問い合わせ一覧
- 新規登録
- ステータス更新
- FAQ候補管理
- 集計・CSV出力

依頼者向けには、以下の画面を想定します。

- 依頼者向け確認

ただし、Ver.2では依頼者向け確認画面はデモ用です。
本格運用では、ログイン認証と権限管理が必要です。

## 4. 要対応アラート画面

### 4.1 目的

要対応アラート画面は、対応漏れや期限超過につながりやすい問い合わせを優先的に確認するための画面です。
管理部担当者が、当日確認すべき問い合わせを素早く把握できるようにします。

### 4.2 表示するアラート

| アラート種別   | 条件                                                 |
| -------------- | ---------------------------------------------------- |
| 期限超過       | 未完了、かつ希望期限が今日より前                     |
| 本日期限       | 未完了、かつ希望期限が今日                           |
| 期限間近       | 未完了、かつ希望期限が明日まで                       |
| 担当者未設定   | 担当者が空欄                                         |
| 情報待ち長期化 | ステータスが情報待ち、かつ受付日から一定日数以上経過 |

### 4.3 KPI表示

画面上部に以下を表示します。

| KPI                | 内容                                     |
| ------------------ | ---------------------------------------- |
| アラート対象件数   | 何らかのアラートに該当する問い合わせ件数 |
| 期限超過件数       | 期限超過の問い合わせ件数                 |
| 本日期限件数       | 希望期限が本日の問い合わせ件数           |
| 期限間近件数       | 希望期限が近い問い合わせ件数             |
| 担当者未設定件数   | 担当者が設定されていない問い合わせ件数   |
| 情報待ち長期化件数 | 情報待ちが長期化している問い合わせ件数   |

### 4.4 絞り込み項目

| フィルタ     | 内容                                                               |
| ------------ | ------------------------------------------------------------------ |
| アラート種別 | すべて、期限超過、本日期限、期限間近、担当者未設定、情報待ち長期化 |

### 4.5 表示項目

| 項目           | 内容           |
| -------------- | -------------- |
| `request_id`   | 問い合わせID   |
| `request_date` | 受付日         |
| `requester`    | 依頼者         |
| `department`   | 部署           |
| `category`     | カテゴリ       |
| `priority`     | 優先度         |
| `due_date`     | 希望期限       |
| `assignee`     | 担当者         |
| `status`       | ステータス     |
| `alert_type`   | アラート種別   |
| `detail`       | 問い合わせ内容 |

## 5. 問い合わせ一覧画面

### 5.1 目的

問い合わせ一覧画面は、登録済みの問い合わせを一覧で確認するための画面です。
部署、カテゴリ、担当者、ステータス、優先度、期限超過などで絞り込み、問い合わせ全体の状況を把握します。

### 5.2 表示項目

| 項目               | 内容               |
| ------------------ | ------------------ |
| `request_id`       | 問い合わせID       |
| `request_date`     | 受付日             |
| `requester`        | 依頼者             |
| `department`       | 部署               |
| `channel`          | 受付経路           |
| `category`         | カテゴリ           |
| `subcategory`      | 小分類             |
| `priority`         | 優先度             |
| `due_date`         | 希望期限           |
| `assignee`         | 担当者             |
| `status`           | ステータス         |
| `overdue_flag`     | 期限超過           |
| `detail`           | 問い合わせ内容     |
| `additional_info`  | カテゴリ別追加情報 |
| `response_summary` | 対応内容           |

### 5.3 フィルタ項目

| フィルタ     | 対象                  |
| ------------ | --------------------- |
| 部署         | `department`          |
| カテゴリ     | `category`            |
| 担当者       | `assignee`            |
| ステータス   | `status`              |
| 優先度       | `priority`            |
| 期限超過のみ | `overdue_flag = True` |

### 5.4 KPI表示

画面上部に以下を表示します。

| KPI          | 内容                       |
| ------------ | -------------------------- |
| 全件数       | 登録されている問い合わせ数 |
| 未完了件数   | ステータスが完了以外の件数 |
| 期限超過件数 | 未完了、かつ期限切れの件数 |
| 本日受付件数 | 受付日が本日の件数         |

絞り込みを行った場合、KPIも絞り込み後の件数に合わせて表示します。

## 6. 新規登録画面

### 6.1 目的

新規登録画面は、管理部が新しい問い合わせを登録するための画面です。
Ver.2では、カテゴリ選択に応じて追加項目を表示し、初回登録時点で必要情報を集めやすくします。

### 6.2 入力項目

| 項目              | 必須 | 初期値・選択肢             |
| ----------------- | ---- | -------------------------- |
| `requester`       | 必須 | なし                       |
| `department`      | 必須 | configから選択             |
| `channel`         | 必須 | フォーム                   |
| `category`        | 必須 | configから選択             |
| `subcategory`     | 任意 | なし                       |
| `detail`          | 必須 | なし                       |
| `missing_info`    | 任意 | なし                       |
| `priority`        | 必須 | 中                         |
| `due_date`        | 必須 | 今日から数日後             |
| `assignee`        | 任意 | 未設定                     |
| `status`          | 必須 | 未対応                     |
| `additional_info` | 任意 | カテゴリ別入力項目から生成 |

### 6.3 カテゴリ別入力フォーム

カテゴリを選択すると、カテゴリに応じた追加項目を表示します。

| カテゴリ         | 追加項目                                                 |
| ---------------- | -------------------------------------------------------- |
| PC・システム     | PC管理番号、発生時刻、エラー内容、再起動有無             |
| アカウント・権限 | 対象システム、対象フォルダ・対象機能、必要な権限、承認者 |
| 勤怠・労務       | 対象日、修正前時刻、修正後時刻、理由                     |
| 経費・請求       | 金額、取引先、対象月、添付書類有無                       |
| 備品・設備       | 備品名・設備名、数量、利用目的、希望日                   |

入力されたカテゴリ別追加情報は、`additional_info` に複数行テキストとして保存します。

### 6.4 登録時の処理

登録時には以下を行います。

- `request_id` を自動発行する
- `request_date` を登録する
- `request_time` を登録する
- `completed_date` は空欄にする
- `management_minutes` は初期値0または空欄にする
- `actual_response_minutes` は初期値0または空欄にする
- `faq_candidate` は0にする
- `requester_visible` は1にする
- `additional_info` にカテゴリ別追加情報を保存する

## 7. ステータス更新画面

### 7.1 目的

ステータス更新画面は、管理部が既存問い合わせの対応状況を更新するための画面です。
問い合わせ内容とカテゴリ別追加情報を確認した上で、担当者、ステータス、対応内容、作業時間を更新します。

### 7.2 表示項目

更新対象を選択した後、以下を表示します。

| 項目               | 内容               |
| ------------------ | ------------------ |
| `request_id`       | 問い合わせID       |
| `requester`        | 依頼者             |
| `department`       | 部署               |
| `category`         | カテゴリ           |
| `subcategory`      | 小分類             |
| `detail`           | 問い合わせ内容     |
| `missing_info`     | 不足情報・確認事項 |
| `additional_info`  | カテゴリ別追加情報 |
| `due_date`         | 希望期限           |
| `status`           | 現在のステータス   |
| `response_summary` | 現在の対応内容     |

`additional_info` がない場合は、カテゴリ別追加情報が登録されていない旨を表示します。

### 7.3 更新項目

| 項目                      | 内容                                     |
| ------------------------- | ---------------------------------------- |
| `assignee`                | 担当者                                   |
| `status`                  | 未対応、対応中、情報待ち、承認待ち、完了 |
| `response_summary`        | 対応内容                                 |
| `record_issue`            | 記録・管理上の問題                       |
| `completed_date`          | 完了日                                   |
| `management_minutes`      | 管理作業時間                             |
| `actual_response_minutes` | 実対応時間                               |

### 7.4 完了時の処理

ステータスを完了にした場合は、完了日を保存します。ステータスが完了以外の場合は、完了日は保存しません。
完了済みの問い合わせを完了以外に戻した場合は、完了日をクリアします。
対応日数は、集計処理やTableau出力時に計算します。

## 8. FAQ候補管理画面

### 8.1 目的

FAQ候補管理画面は、完了済み問い合わせの中から、FAQ化できそうな問い合わせを候補として登録・更新・解除するための画面です。
Ver.2では、FAQ公開ページまでは作らず、FAQ候補の蓄積を対象とします。

### 8.2 対象データ

FAQ候補管理画面では、ステータスが完了の問い合わせを対象とします。

```text
status == "完了"
```

### 8.3 表示項目

問い合わせを選択した後、以下を表示します。

| 項目               | 内容                |
| ------------------ | ------------------- |
| `request_id`       | 問い合わせID        |
| `category`         | カテゴリ            |
| `subcategory`      | 小分類              |
| `detail`           | 問い合わせ内容      |
| `additional_info`  | カテゴリ別追加情報  |
| `response_summary` | 対応内容            |
| `completed_date`   | 完了日              |
| `faq_candidate`    | 現在FAQ候補かどうか |

表示順は以下とします。

```text id="lkdghm"
問い合わせ内容
カテゴリ別追加情報
対応内容
完了日
FAQ候補状態メッセージ
```

`additional_info` がない場合は、カテゴリ別追加情報が登録されていない旨を表示します。

FAQ候補状態は、以下のように表示します。

| 状態            | 表示                                        |
| --------------- | ------------------------------------------- |
| FAQ候補である   | この問い合わせは現在FAQ候補です。           |
| FAQ候補ではない | この問い合わせはまだFAQ候補ではありません。 |

### 8.4 入力項目

| 項目         | 内容                            |
| ------------ | ------------------------------- |
| `faq_title`  | FAQとして整理する場合のタイトル |
| `faq_answer` | 依頼者向けに整理した回答案      |

### 8.5 操作

| 操作                     | 内容                                                   |
| ------------------------ | ------------------------------------------------------ |
| FAQ候補として保存・更新  | `faq_candidate` を1にし、FAQタイトル・回答案を保存する |
| FAQ候補から外す          | `faq_candidate` を0にする                              |
| FAQ候補CSVをダウンロード | FAQ候補一覧をCSVとして出力する                         |

FAQ候補から外しても、FAQタイトル・FAQ回答案は下書きとして残します。

## 9. 依頼者向け確認画面

### 9.1 目的

依頼者向け確認画面は、依頼者が自分の問い合わせ状況を確認するための画面です。
Ver.2ではデモ用として実装し、本格的なログイン認証・権限管理は今後の課題とします。

### 9.2 検索方法

| 検索方法           | 内容                            |
| ------------------ | ------------------------------- |
| 問い合わせIDで検索 | `request_id` を入力して検索する |
| 依頼者名で検索     | `requester` を入力して検索する  |

検索UIはフォーム化し、Enterキーまたは検索ボタンで検索できるようにします。

### 9.3 表示対象

`requester_visible` が1の問い合わせを表示対象とします。

```text id="i68cg0"
requester_visible == 1
```

`requester_visible` が0の問い合わせは、依頼者向け確認画面には表示しません。

### 9.4 表示項目

| 項目               | 内容                       |
| ------------------ | -------------------------- |
| `request_id`       | 問い合わせID               |
| `request_date`     | 受付日                     |
| `requester`        | 依頼者                     |
| `department`       | 部署                       |
| `category`         | カテゴリ                   |
| `subcategory`      | 小分類                     |
| `detail`           | 問い合わせ内容             |
| `additional_info`  | カテゴリ別追加情報         |
| `status`           | ステータス                 |
| `assignee`         | 担当者                     |
| `due_date`         | 希望期限                   |
| `completed_date`   | 完了日                     |
| `response_summary` | 管理部からの回答・対応内容 |

### 9.5 表示しない項目

依頼者向け確認画面では、以下の内部管理項目は表示しません。

| 項目                      | 理由                        |
| ------------------------- | --------------------------- |
| `management_minutes`      | 管理部内部の作業時間のため  |
| `actual_response_minutes` | 管理部内部の作業時間のため  |
| `record_issue`            | 管理上の内部メモのため      |
| `faq_candidate`           | FAQ管理用の内部フラグのため |
| `faq_title`               | FAQ候補管理用の下書きのため |
| `faq_answer`              | FAQ候補管理用の下書きのため |

| `requester_visible_int`   | 依頼者向け画面に表示するか           |
| `requester_visible_label` | 依頼者向け表示区分                   |

## 11. 画面設計上の方針

本システムでは、以下の方針で画面を設計します。

| 方針                         | 内容                                                     |
| ---------------------------- | -------------------------------------------------------- |
| 管理部の確認負担を減らす     | 一覧、アラート、集計で状況を把握しやすくする             |
| 入力ミスを減らす             | 部署、カテゴリ、ステータスなどは選択式にする             |
| 確認往復を減らす             | カテゴリ別入力フォームで必要情報を初回登録時に集める     |
| 対応漏れを防ぐ               | 要対応アラートで期限超過や担当者未設定を見つけやすくする |
| ナレッジ化につなげる         | 完了済み問い合わせをFAQ候補として整理できるようにする    |
| 依頼者の確認負担を減らす     | 依頼者向け確認画面で問い合わせ状況を確認できるようにする |
| 本格運用との差分を明確にする | 認証・権限管理はVer.3以降の課題として明示する            |

| 通知設定画面       | メール・Slack通知条件を設定する                   |
| 添付ファイル画面   | スクリーンショットや証憑資料を登録する            |
| 操作履歴画面       | 誰がいつ何を更新したかを確認する                  |
| ダッシュボード画面 | アラート、FAQ候補、作業時間、対応状況を可視化する |

## 13. まとめ

Ver.2では、Ver.1の4画面構成を拡張し、要対応アラート、FAQ候補管理、依頼者向け確認を追加しました。また、新規登録画面にはカテゴリ別入力フォームを追加し、集計・CSV出力画面にはVer.2追加集計を反映しています。
これにより、問い合わせを記録・管理するだけでなく、対応漏れ防止、確認往復削減、FAQ候補蓄積、依頼者側の状況確認まで含めた画面構成になっています。
