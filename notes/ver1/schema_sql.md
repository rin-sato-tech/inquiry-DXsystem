# schema.sqlファイルを作る意味

`schema.sql` は、ひと言でいうと SQLiteデータベースの設計図です。
今回のシステムでは、`data/inquiry.db` というDBファイルを作りますが、その中にどんなテーブルを作るか、どんな列を持たせるか、どの列を検索しやすくするかを定義しているのが `schema.sql` です。

## 1. `schema.sql`の役割

今回の `schema.sql` は、主に以下を定義しています。

```text
1. inquiriesテーブルを作る
2. 各カラム名を決める
3. 各カラムの型を決める
4. 必須項目を決める
5. 主キーを決める
6. 検索用インデックスを作る
```

つまり、問い合わせデータを保存するための「器」を作っています。

## 2. なぜ`db.py`に直接書かないのか

もちろん、Pythonの `db.py` の中にSQL文を直接書くこともできます。
たとえば、

```python
conn.execute("""
CREATE TABLE inquiries (...)
""")
```

のようにできます。しかし、`schema.sql` として分けるとメリットがあります。

| 分ける理由                   | 内容                                            |
| ---------------------------- | ----------------------------------------------- |
| DB構造が見やすい             | テーブル定義だけを確認できる                    |
| 再作成しやすい               | DBを消しても同じ構造で作り直せる                |
| Pythonコードが整理される     | `db.py` は処理、`schema.sql` は設計に分けられる |
| ポートフォリオで説明しやすい | DB設計を明確に示せる                            |
| 保守しやすい                 | カラム追加・変更時に場所が分かりやすい          |

つまり、schema.sql は DB設計を独立したファイルとして見える化するために置いています。

## 3. 今回の処理の流れ

今回のDB作成処理はこうなっています。

```text
schema.sql
↓
src/db.py の init_db()
↓
SQLiteに inquiries テーブルを作成
↓
data/inquiry.db が使えるようになる
```

具体的には、`src/db.py` のこの部分で読み込んでいます。

```python
schema_sql = SCHEMA_PATH.read_text(encoding="utf-8")
conn.executescript(schema_sql)
```

つまり、Python側は `schema.sql` を読み込んで、そこに書かれているSQLをSQLiteに実行させています。

## 4. `config`との違い

`config` と `schema.sql` はどちらも「設定っぽい」ですが、役割が違います。

|ファイル|役割|
|---|---|
|`schema.sql`|データベースの構造を決める|
|`config/*.csv`|画面で使う選択肢を決める|

たとえば、`status という列を持つ`と決めるのが `schema.sql`。`statusの選択肢は、未対応・対応中・情報待ち・承認待ち・完了`と決めるのが `config/status_master.csv` です。

## 5. ポートフォリオ上の説明

面接やREADMEでは、こう説明できます。

>schema.sql には、問い合わせ情報を保存する inquiries テーブルの構造を定義しました。
問い合わせID、受付日、依頼者、カテゴリ、担当者、ステータス、期限、対応内容、作業時間などを保存できるようにし、ステータス・カテゴリ・担当者・期限など、検索や絞り込みでよく使う項目にはインデックスを設定しています。
DB構造をPythonコードから分離することで、データベース設計を確認しやすくし、再作成や保守をしやすくしています。

## 結論

`schema.sql` は、SQLiteデータベースの設計図です。
今回でいうと、問い合わせを保存する `inquiries` テーブルのカラム、型、必須項目、主キー、インデックスを定義しています。
`config` が「選択肢の管理」だとすれば、`schema.sql` は「データの入れ物の構造管理」です。
